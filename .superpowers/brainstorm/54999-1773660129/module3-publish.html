Module 3: Publish Engine — Playwright Browser Automation

Automated publishing to Xiaohongshu & Douyin via browser control. New file: src/publish-engine.ts

Publishing Flow
Publish Lifecycle
① User clicks "Publish" on ready work
  → selects platforms: [xiaohongshu, douyin]
  → optional: schedule time or publish now
② Backend checks login state
  → load saved cookies from ~/.skill-evolver/auth/
  → if expired → prompt user to re-login
③ Launch Playwright browser (per platform)
  → chromium.launchPersistentContext(userDataDir)
  → persistent context = cookies survive restarts
④ Platform adapter executes publish
  → navigate to publish page
  → fill title, content, upload media
  → add tags/topics
  → click publish
⑤ Verify & capture proof
  → wait for success confirmation
  → screenshot → save to work assets
  → extract post URL
⑥ Update work store + EverMemOS
  → status → published, save postUrl
  → write memory: content + platform + time
Login Management
Strategy: Persistent Browser Context
✓ Use launchPersistentContext() with userDataDir
✓ Cookies + localStorage persist across launches
✓ User logs in once via visible browser window
✓ Subsequent publishes reuse session automatically
First-time Login Flow:
1. Open visible browser: headless: false
2. Navigate to login page
3. User scans QR code / enters credentials
4. Detect login success (URL change or cookie)
5. Save context → future launches headless

Error Handling & Safety

Failure Scenarios
• Login expired → prompt user, don't auto-retry
• Upload timeout → retry once, then mark failed
• Platform UI changed → adapter throws, work stays "ready"
• Network error → retry with backoff (max 3)
• Content rejected by platform → capture error screenshot
Safety Measures
• Visible browser for first login (no credential storage)
• Screenshot every publish for audit trail
• User must confirm before publishing (no auto-publish without consent)
• Rate limiting: max 5 publishes per platform per day
• Dry-run mode: fill form but don't click submit
