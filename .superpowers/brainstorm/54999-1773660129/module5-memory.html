Memory Architecture
Memory Categories for AutoViral
1. Creation Memory (episodic)
What: 每次创作的选题、风格、用户反馈
When: 每步完成 + work 完成时写入
Format: group_id: "autoviral_work_{workId}"
Example: "为用户创作了关于AI工具的短视频，选题角度是'3个改变工作效率的AI工具'，用户要求更接地气的标题，修改为'打工人必备的3个AI神器'"
2. Performance Memory (episodic)
What: 发布后的数据表现和归因分析
When: 指标采集后写入（关键节点：24h、72h、7d）
Format: group_id: "autoviral_metrics_{workId}"
Example: "短视频'打工人必备的3个AI神器'在抖音24h数据：播放4.5万，点赞3200，完播率42%。表现高于该账号均值2.3倍，hook在前1.5秒的策略有效。"
3. Platform Rules (profile)
What: 从成功/失败中学到的平台规则
When: 周度复盘 Agent 分析后写入
Format: category: "platform_rules"
Example: "小红书图文笔记封面带人脸的点击率高30%；标题使用'数字+痛点'格式表现最好；下午6-8点发布获得最多初始流量"
4. Style Profile (profile)
What: 用户的内容风格画像，持续演进
When: 每5个 work 后 Agent 重新评估
Format: category: "style_profile"
Example: "用户偏好：节奏快、信息密度高、实用导向；常用剪辑风格：卡点、弹幕字幕、数据可视化；语气：轻松幽默但不油腻"
5. Competitor Insights (episodic)
What: 竞品内容策略和表现分析
When: 竞品数据采集后写入
Format: group_id: "autoviral_competitor_{name}"
Example: "竞品@科技范儿本周发3条：AI工具测评、效率app推荐、新品开箱。AI测评类播放最高(12万)，开箱类表现一般(2万)。其hook策略：先展示效果再讲方法。"
Read/Write Flow
READ — Before Creation
1. searchMemories({query: workTopic, method: "hybrid", top_k: 10})
   → find related past creations + their performance
2. searchMemories({query: "style_profile", memory_types: ["profile"]})
   → get latest style preferences
3. searchMemories({query: "platform_rules " + platform})
   → get learned platform rules
4. Compose into memory context block
5. Inject into CLI session initial prompt
WRITE — After Key Events
1. Pipeline step completed → addMemory(creation)
2. Work published → addMemory(publish event)
3. Metrics at 24h/72h/7d → addMemory(performance)
4. Weekly review → addMemory(rules + style)
5. Competitor scraped → addMemory(competitor insight)
Memory Context Injection
// Injected into CLI prompt at session start: ## 你的创作记忆 ### 风格画像 {style_profile from EverMemOS} ### 平台规则 ({platform}) {platform_rules from EverMemOS} ### 相关历史创作 {top 3 related past works + their metrics} ### 竞品动态 {latest competitor insights} // This gives the Agent full context about: // - How user likes to create // - What works on this platform // - Past wins and losses // - What competitors are doing
Weekly Review Agent
Cron: weekly (Sunday, configurable)
Uses existing executor (evo-skill pattern)
1. Search all creation + performance memories from past 7 days
2. Agent analyzes: what content performed best/worst?
3. Agent generates updated profile memories:
  • Style profile refresh
  • Platform rules update
  • Suggested topics for next week
4. Write to EverMemOS as profile type
5. Generate report → shown on Analytics page
