Module 4: Data Collector — Metric Scraping Engine

Playwright-based periodic data collection. Replaces all mock data with real metrics. New file: src/data-collector.ts

Collection Architecture
Two Collection Modes
A. Post Metrics — Track Your Content
Trigger: Cron (every 4h for first 48h, then daily for 7d, then weekly)
Target: Each published work's postUrl on each platform
Collects:
  • views / plays
  • likes / favorites
  • comments count
  • shares / reposts
  • follower delta (if available)
Stores: work.yaml → platforms[].metrics + metrics history
Memory: → EverMemOS episodic (performance record)
B. Trend Data — Discover What's Hot
Trigger: Cron (every 6h, configurable)
Target: Platform trending/explore pages
Collects:
  • Trending videos (title, views, likes, creator)
  • Hot topics / hashtags (post count, trend direction)
  • Competitor content updates (if configured)
Stores: ~/.skill-evolver/trends/{platform}/{date}.yaml
Memory: → EverMemOS episodic (trend snapshot)
Smart Scheduling
Post Metrics — Decay Schedule
0-48h every 4 hours (12 collections)
2-7 days once daily (5 collections)
7-30 days weekly (~3 collections)
30d+ stop (content lifecycle over)
Total ~20 scrapes per work per platform. Minimizes requests.
Trend Data — Fixed Interval
Default: every 6h (configurable in config.yaml)
4 collections/day × 2 platforms = 8 scrapes/day

New API Routes

GET /api/trends/:platform # latest trend snapshot (xiaohongshu | douyin) GET /api/trends/:platform/history # trend snapshots over time GET /api/analytics # aggregated: total views/likes, avg growth, top works GET /api/analytics/work/:id # per-work growth curves POST /api/collector/trigger # manually trigger a collection run GET /api/collector/status # next scheduled run, last run stats GET /api/competitors # list tracked competitor profiles POST /api/competitors # add competitor to track {platform, profileUrl}
