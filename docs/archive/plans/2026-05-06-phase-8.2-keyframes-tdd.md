# Phase 8.2 — Keyframes (transform / opacity / volume curves) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Today every clip has *static* values for `scale / x / y / rotation` (VideoClip + OverlayClip transforms), `opacity` (OverlayClip), and `volume` (AudioClip). Phase 8.2 turns each of those numeric properties into an *optional* keyframe curve. A clip carries a flat `keyframes?: Keyframe[]` array (each entry tags its property + time + value + easing); the Remotion track renderers consume that array via a pure `interpolateProperty(keyframes, property, currentTime)` helper at every frame; the Inspector grows a `KeyframePanel` (table-style v1) that lets the user add / edit / delete keyframes for the selected clip.

The acceptance video is a `VideoClip` with two `scale` keyframes — `t=0 v=1, t=2 v=2` — that visibly zooms from 1× to 2× over the first 2 seconds in *both* the live preview and the exported MP4. No legacy clip data needs migration: `keyframes` is `optional()`, every existing comp continues to load and render with the static fallback.

**Architecture:** Phase 8.2 is intentionally *thin*. The flat-array schema (D2) plus a single pure helper (`interpolateProperty`) means the track renderers each gain ~3 lines of code to call `interpolateProperty(c.keyframes, "scale", localSec) ?? c.transforms.scale`. The store grows three actions (`addKeyframe`, `removeKeyframe`, `updateKeyframe`) that mutate the same clip the existing `updateClip` already targets — no new selection model, no new collection. The Inspector panel is a list/table of rows: one row per keyframe; rows sort by `(property, time)`. Graph editing (drag handles on a curve) is explicitly deferred to a Phase 8.2.5 follow-up.

**Tech Stack:**
- Schema: `zod` (already in `src/shared/composition.ts`); `KeyframeSchema` slots into the same file alongside `TransformsSchema` / `FiltersSchema`.
- Pure helper: zero deps, plain TS in `src/shared/keyframes.ts` so both Remotion render path (`src/server/remotion-renderer.ts`) and the Studio web UI can consume it.
- Remotion: existing `useCurrentFrame()` + `useVideoConfig()`. The renderers already call Remotion's own `interpolate` for text animations — we are not replacing that; `interpolateProperty` is a separate helper because keyframe arrays carry per-segment easing whereas Remotion's `interpolate(frame, [a,b], [v0,v1])` only takes a global one.
- Store: existing `zustand` + `immer`.
- UI: existing aesthetic tokens (`var(--accent)`, mono labels, glass-border rows). No new lib.
- Tests: Vitest + Testing Library, same conventions as Phase 5/6/7.

## Task 8.2.A — Schema extension + interpolateProperty pure helper

**Goal:** Land the contract. After this task, `Composition` validates with `keyframes?: Keyframe[]` on each visual/audio clip; `interpolateProperty(kfs, property, t)` computes the animated value at any time; ~10 tests cover the helper exhaustively.

**Pitfalls:**
- The master-plan §8.2 sketch uses `prop`/`t`/`v`/`ease` keys with hyphen-cased easings (`"ease-in"`). We intentionally diverge to `property`/`time`/`value`/`easing` (camelCase, parallel to the rest of `composition.ts`) and `easeIn` / `easeOut` / `easeInOut` (also camelCase, parallel to Remotion's `Easing`). The new `KeyframeSchema` is the canonical contract. The master plan is the parent design doc; this plan is the executable spec.
- Floating-point time math bites. Use `EPSILON = 1e-4` for time equality (D4). Vitest's default tolerance won't help — write `expect(Math.abs(a - b)).toBeLessThan(1e-6)` for value assertions where easing maths produces tiny irrational numbers.
- Easing implementations must match Remotion's `Easing.bezier` outputs to within a tight tolerance (~1e-3) so the *preview* curve matches the *export* curve. Cheapest path: just use the same cubic-bezier coefficients Remotion uses (`Easing.bezier(0.42, 0, 1, 1)` for easeIn etc.) and inline-compute via Newton iteration.
- `Keyframe.property` enum includes `"volume"` — VideoClip / OverlayClip schemas accept it (zod won't reject it at schema level), but the runtime helpers / renderers ignore unsupported (clip × property) pairs. Document this in a JSDoc on `interpolateProperty`.

### Step 3: Implement `keyframes.ts`

- [ ] Create `src/shared/keyframes.ts`:

  ```ts
  import type { Keyframe, KeyframeEasing, KeyframeProperty } from "./composition";

  export const KEYFRAME_TIME_EPSILON = 1e-4;

  /** Pure cubic-bezier evaluation matching Remotion's Easing.bezier outputs. */
  function bezier(p1x: number, p1y: number, p2x: number, p2y: number, x: number): number {
    // Newton-Raphson on the X axis, then evaluate Y at the matching t.
    // (Standard impl; see WebKit cubic-bezier source.)
    // ... ~30 lines ...
  }

  function applyEasing(easing: KeyframeEasing, t: number): number {
    switch (easing) {
      case "linear":     return t;
      case "easeIn":     return bezier(0.42, 0, 1, 1, t);
      case "easeOut":    return bezier(0, 0, 0.58, 1, t);
      case "easeInOut":  return bezier(0.42, 0, 0.58, 1, t);
    }
  }

  /**
   * Returns the interpolated value for `property` at clip-local time `currentTime`,
   * or `null` if the array contains no keyframe for that property.
   * - Out-of-range times clamp to the nearest endpoint (D3).
   * - The easing of the *outgoing* keyframe (the one at the segment's start) controls the segment.
   * - Volume on a VideoClip is structurally allowed but renderers ignore it (D5).
   */
  export function interpolateProperty(
    keyframes: readonly Keyframe[] | undefined,
    property: KeyframeProperty,
    currentTime: number,
  ): number | null {
    if (!keyframes || keyframes.length === 0) return null;
    const filtered = keyframes
      .filter((k) => k.property === property)
      .slice()
      .sort((a, b) => a.time - b.time);
    if (filtered.length === 0) return null;
    if (currentTime <= filtered[0].time) return filtered[0].value;
    const last = filtered[filtered.length - 1];
    if (currentTime >= last.time) return last.value;
    // find segment
    for (let i = 0; i < filtered.length - 1; i++) {
      const a = filtered[i];
      const b = filtered[i + 1];
      if (currentTime >= a.time && currentTime <= b.time) {
        const tNorm = (currentTime - a.time) / (b.time - a.time);
        const eased = applyEasing(a.easing, tNorm);
        return a.value + (b.value - a.value) * eased;
      }
    }
    return last.value; // unreachable, but keeps TS happy
  }

  /** Idempotent insert: replaces existing entry at (property, time±EPSILON). D4. */
  export function addOrReplaceKeyframe(
    keyframes: readonly Keyframe[] | undefined,
    next: Keyframe,
  ): Keyframe[] {
    const arr = keyframes ? keyframes.slice() : [];
    const idx = arr.findIndex(
      (k) => k.property === next.property && Math.abs(k.time - next.time) < KEYFRAME_TIME_EPSILON,
    );
    if (idx >= 0) {
      arr[idx] = next;
    } else {
      arr.push(next);
      arr.sort((a, b) => (a.property === b.property ? a.time - b.time : a.property.localeCompare(b.property)));
    }
    return arr;
  }
  ```

### Step 6: Bezier reference implementation

The `bezier(p1x, p1y, p2x, p2y, x)` helper is small but the math is fiddly. Drop in this canonical implementation (mirrors WebKit / Remotion):

```ts
function bezier(p1x: number, p1y: number, p2x: number, p2y: number, x: number): number {
  // Cubic Bezier defined by (0,0), (p1x,p1y), (p2x,p2y), (1,1).
  // Compute t for the given x via Newton-Raphson, then evaluate y(t).
  if (x <= 0) return 0;
  if (x >= 1) return 1;

  const cx = 3 * p1x;
  const bx = 3 * (p2x - p1x) - cx;
  const ax = 1 - cx - bx;

  const cy = 3 * p1y;
  const by = 3 * (p2y - p1y) - cy;
  const ay = 1 - cy - by;

  const sampleCurveX = (t: number) => ((ax * t + bx) * t + cx) * t;
  const sampleCurveY = (t: number) => ((ay * t + by) * t + cy) * t;
  const sampleCurveDerivativeX = (t: number) => (3 * ax * t + 2 * bx) * t + cx;

  let t = x;
  for (let i = 0; i < 8; i++) {
    const x2 = sampleCurveX(t) - x;
    if (Math.abs(x2) < 1e-7) return sampleCurveY(t);
    const d = sampleCurveDerivativeX(t);
    if (Math.abs(d) < 1e-7) break;
    t = t - x2 / d;
  }
  // Bisection fallback
  let lo = 0, hi = 1;
  while (lo < hi) {
    const x2 = sampleCurveX(t) - x;
    if (Math.abs(x2) < 1e-7) return sampleCurveY(t);
    if (x2 > 0) hi = t; else lo = t;
    t = (hi + lo) / 2;
  }
  return sampleCurveY(t);
}
```

```tsx
import { Sequence, useVideoConfig, Img, useCurrentFrame } from "remotion";
import type { OverlayClip, Track } from "../../types";
import { interpolateProperty } from "@shared/keyframes";

function OverlayClipRenderer({ clip }: { clip: OverlayClip }) {
  const { fps } = useVideoConfig();
  const frame = useCurrentFrame();
  const localSec = frame / fps;
  const opacity  = interpolateProperty(clip.keyframes, "opacity",  localSec) ?? clip.opacity;
  const scale    = interpolateProperty(clip.keyframes, "scale",    localSec) ?? 1;
  const x        = interpolateProperty(clip.keyframes, "x",        localSec) ?? 0;
  const y        = interpolateProperty(clip.keyframes, "y",        localSec) ?? 0;
  const rotation = interpolateProperty(clip.keyframes, "rotation", localSec) ?? 0;
  return (
    <Img
      src={clip.src}
      style={{
        position: "absolute",
        left: `${clip.position.xPct}%`,
        top: `${clip.position.yPct}%`,
        width: `${clip.position.wPct}%`,
        height: `${clip.position.hPct}%`,
        opacity,
        transform: `translate(${x}px, ${y}px) rotate(${rotation}deg) scale(${scale})`,
        transformOrigin: "center center",
      }}
    />
  );
}

export function OverlayTrackRenderer({ track }: { track: Track }) {
  const { fps } = useVideoConfig();
  if (track.hidden) return null;
  return (
    <>
      {(track.clips as OverlayClip[]).map((c) => (
        <Sequence
          key={c.id}
          from={Math.round(c.trackOffset * fps)}
          durationInFrames={Math.max(1, Math.round(c.duration * fps))}
        >
          <OverlayClipRenderer clip={c} />
        </Sequence>
      ))}
    </>
  );
}
```

Note: OverlayClip's static defaults for `scale / x / y / rotation` are `1, 0, 0, 0` — no Transforms field today. We treat absent keyframes as identity transform, which matches the prior behavior.

### Step 4: KeyframePanel.tsx — recommended structure

```tsx
import { useMemo, useState } from "react";
import { useComposition } from "../../store";
import type { Clip, Keyframe, KeyframeProperty, KeyframeEasing } from "../../types";

const EASINGS: KeyframeEasing[] = ["linear", "easeIn", "easeOut", "easeInOut"];

function propertiesForClip(kind: Clip["kind"]): KeyframeProperty[] {
  switch (kind) {
    case "video":   return ["scale", "x", "y", "rotation"];
    case "overlay": return ["scale", "x", "y", "rotation", "opacity"];
    case "audio":   return ["volume"];
    case "text":    return [];
  }
}

export function KeyframePanel() {
  const comp = useComposition((s) => s.comp);
  const selection = useComposition((s) => s.selection);
  const addKeyframe = useComposition((s) => s.addKeyframe);
  const removeKeyframe = useComposition((s) => s.removeKeyframe);
  const updateKeyframe = useComposition((s) => s.updateKeyframe);
  const [adding, setAdding] = useState(false);

  const clip = useMemo<Clip | null>(() => {
    if (!comp || !selection) return null;
    for (const t of comp.tracks) {
      const c = (t.clips as Clip[]).find((c) => c.id === selection);
      if (c) return c;
    }
    return null;
  }, [comp, selection]);

  if (!clip) return <Empty>Select a clip in the timeline to add keyframes</Empty>;
  if (clip.kind === "text") return <Empty>Text clips use the animation enum, not keyframes</Empty>;

  const kfs = ((clip as { keyframes?: Keyframe[] }).keyframes ?? []).map((kf, i) => ({ kf, idx: i }));
  const grouped = new Map<KeyframeProperty, Array<{ kf: Keyframe; idx: number }>>();
  for (const entry of kfs) {
    const arr = grouped.get(entry.kf.property) ?? [];
    arr.push(entry);
    grouped.set(entry.kf.property, arr);
  }
  for (const arr of grouped.values()) arr.sort((a, b) => a.kf.time - b.kf.time);

  const props = propertiesForClip(clip.kind);

  return (
    <section className="keyframe-panel" style={{ display: "flex", flexDirection: "column", gap: 10 }}>
      <Header onAddClick={() => setAdding(true)} />
      {adding && (
        <AddForm
          properties={props}
          defaultValueFor={(p) => defaultStaticValue(clip, p)}
          onSubmit={(kf) => { addKeyframe(clip.id, kf); setAdding(false); }}
          onCancel={() => setAdding(false)}
        />
      )}
      {grouped.size === 0 && !adding && (
        <Empty>No keyframes yet — click "Add keyframe" to author your first one</Empty>
      )}
      {[...grouped.entries()].map(([prop, rows]) => (
        <PropertyBlock
          key={prop}
          property={prop}
          rows={rows}
          onRemove={(idx) => removeKeyframe(clip.id, idx)}
          onUpdate={(idx, patch) => updateKeyframe(clip.id, idx, patch)}
        />
      ))}
    </section>
  );
}

function defaultStaticValue(clip: Clip, prop: KeyframeProperty): number {
  if (clip.kind === "video") {
    if (prop === "scale")    return clip.transforms.scale;
    if (prop === "x")        return clip.transforms.x;
    if (prop === "y")        return clip.transforms.y;
    if (prop === "rotation") return clip.transforms.rotation;
  }
  if (clip.kind === "overlay") {
    if (prop === "opacity")  return clip.opacity;
    return prop === "scale" ? 1 : 0;
  }
  if (clip.kind === "audio") {
    if (prop === "volume") return clip.volume;
  }
  return 0;
}
```

Subcomponents (`Header`, `AddForm`, `PropertyBlock`, `Empty`) are short — keep them in the same file. Style them with the existing aesthetic tokens (`var(--surface-0)`, `var(--glass-border)`, `var(--font-mono)` for labels, `var(--accent)` for the primary action).
