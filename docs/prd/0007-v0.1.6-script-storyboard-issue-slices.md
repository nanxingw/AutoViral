## S1 · 分镜数据核心（keystone）

- **Type:** AFK
- **覆盖:** 全部下游地基；PRD §4

### What to build

升级潜伏的 `SceneSchema`，让"分镜"成为可写入、可校验、向后兼容的一等数据，并提供意图级 mutation 的共享纯函数（`@shared/composition/ops`，对齐 ADR-009）。这是 UI/CLI/生成三方共同消费的单一事实源，本片只交付数据 + 纯函数 + 单测（无 UI），靠单测即可独立验证。

新增字段（全 optional/有 default）：`narration`(旁白) / `durationSec`(时长) / `shotSize`(景别 enum) / `cameraMovement`(运镜 enum) / `generatedAssetIds`(default []) / `selectedAssetId` / `status`(planned|generated|stale, default planned) / `mdAnchor`。保留现有 `id/order/title/prompt/intent/memberClipIds/memberAssetIds` 不动。ops：`addScene`(内部 mint id + 自动分配 order) / `setSceneProps` / `reorderScenes`(连号) / `linkSceneAssets` / `removeScene`。

### Acceptance criteria

- [ ] `SceneSchema` 含 8 个新字段，全 optional/有 default；0.1.5 旧 work（无 `scenes` key、无新字段）原样 parse —— 加显式断言
- [ ] 新建 `newSceneId()` 原语（mirror 既有 `newTrackId`）
- [ ] `addScene` 内部 mint sceneId + 自动分配 `order = max(现有 order)+1`（空则 0），**调用方/CLI/bridge 不传 order**
- [ ] `reorderScenes` / `removeScene` 后 `order` 强制连续 `0..N-1`（mirror `recompactDisplayOrder`）
- [ ] 单测：每动词 + 连号不变量 + **「addScene 不传 order 仍 `CompositionWriteSchema.parse` 通过」** + 旧 work parse 回归
- [ ] `ops/index` 导出五个动词；非法参数 throw `CompositionOpError{code:4}`

### Blocked by

None — can start immediately.

## S2 · agent / CLI 驱动分镜（首个可演示纵切）

- **Type:** AFK
- **覆盖:** agent user story（CLI 驱动分镜，与人 UI 一致）；PRD §3/§6

### What to build

把 S1 的 ops 接出 bridge 路由 + CLI 动词，让任意 CLI agent 能端到端地增/改/排/删分镜，写进 `composition.yaml scenes[]` 并广播刷新，且 agent 系统提示里"知道"这套能力。这是第一个可演示的纵切：`autoviral scene add` 一条命令即可验证。

scene 写路由逐字 mirror 既有 `/split`（经 `mutateCompositionFor` + 复用 `composition-changed` 广播）。CLI mirror `clip.ts`（parseFlags → 本地校验 → bridgeRequest）。`buildSystemPrompt` 补 scene 动词 + 工作流叙述（先排剧本、再排分镜、生成交 handoff）+ 产物契约补 `plan/script.md`。

### Acceptance criteria

- [ ] 5 个 scene 写路由（add/set/reorder/link/remove），经同一份 `@shared` ops + 复用 `composition-changed` 广播
- [ ] CLI `autoviral scene add/list/set/reorder/link/remove`；本地校验失败 exit 4，4xx 带 `code`
- [ ] `autoviral scene add --title X --intent hook` 真写进 `composition.yaml scenes[]`（集成测试断言 + 广播 fire）
- [ ] `buildSystemPrompt` 含 scene 动词 + 工作流叙述；manual 补 scenes 字段 + 新 recipe `script-to-storyboard`
- [ ] docs-drift 守卫（manual 引用）绿

- S1

## S3 · 用户看见分镜骨架

- **Type:** AFK
- **覆盖:** creator user story（一眼看全片骨架）；PRD §7

### What to build

素材区（AssetSidebar）加第三个 tab「剧本·分镜」，呈现 `comp.scenes` 的只读卡片列表，让用户一眼看全片骨架。既有 work（无分镜）显诚实引导态。本片只做读路径 + tab 壳 + 修 auto-switch 弹走。

### Acceptance criteria

- [ ] 素材区出现第三 tab「剧本·分镜」，label 经 i18n（en/zh 双份）
- [ ] **停在剧本 tab 时点选 timeline clip 不被自动切到 inspector**（修既有 `if(selection)setTab('inspector')`）—— DOM 二确当前 tab 不变
- [ ] 卡片按 `order` 渲染 `comp.scenes`：镜号 / 标题 / intent 徽章 / status 点 / 画面描述摘要 / 旁白 / 时长 / 景别 / 运镜（可选字段有才显）
- [ ] 既有 work（无 scenes、无 script.md）显本地化引导态，**不空白不报错**；`hasScenes`/`hasScript` 独立判定不互相暗示
- [ ] 浏览器可见 + DOM `textContent` 二确（agent 经 S2 建的分镜在面板里看得到）

### Blocked by

- S1（必需）；S2（推荐先有，便于演示真卡片——也可先用空/引导态验 tab 壳）

## S6 · per-work 写锁加固

- **Type:** AFK
- **覆盖:** 不变量 #1 加固（顺带修既有 11+ 动词 lost-update）；PRD §4.4

### What to build

给 `mutateCompositionFor` 加 per-work 写串行化（async mutex），根治无锁 read-modify-write 的 lost-update——这不只服务 scenes，**一处修所有意图动词 + agent 整份 `/comp` PUT** 的并发覆盖。诚实定位为"加固"，不夸大"结构性消灭"。

### Acceptance criteria

- [ ] `mutateCompositionFor` 同一 work 的写串行化（async mutex/写队列）
- [ ] 并发交错单测：两个 per-intent 写（含一个 scene 写）交错触发，断言两端都落盘、无静默丢写
- [ ] 既有动词（clip/track/transition…）回归全绿

### Blocked by

## S7 · 生成 handoff（计划 → 执行）

- **Type:** AFK
- **覆盖:** creator（一幕一幕生成）；PRD §5

### What to build

分镜卡片「生成此幕」把该镜的 prompt（拼接景别/运镜/旁白富化）**交给现有生成流程**（不内嵌驾驶舱、不建异步队列），产物注册进素材库并回链该分镜。改画面描述标脏、可重拍保留候选。**关键：image 生成路径不写 `composition.assets`（仅 video 写），回链前必须先登记 `AssetEntry` 防悬挂引用。**

### Acceptance criteria

- [ ] 卡片「生成此幕」用 prompt（含景别/运镜/旁白）调现有生成流程
- [ ] 产物登记为 `composition.assets` 的 `AssetEntry`（**image 路径补登记**）；据 HTTP 响应 `linkSceneAssets` 回链 `generatedAssetIds`+`selectedAssetId`+`status=generated`
- [ ] 改画面描述后 `status→stale`，卡片显"需重生"
- [ ] 「重拍」append 新 take，可切换 `selectedAssetId`；（可选）image 多候选复用 batch/select（key 换 sceneId）
- [ ] `autoviral scene generate <id>` CLI
- [ ] 浏览器可见：卡片显缩略图 + 已生成态；产物在素材库可见且 `composition.assets` 有此 asset（**无悬挂引用**，DOM/数据二确）

### Blocked by

- S1, S4

## S8 · 多纬度 E2E

- **Type:** AFK（真生成纬度若需 `OPENROUTER_API_KEY`/付费，则该纬度 `ready-for-human`；默认用 stub provider 走 AFK）
- **覆盖:** 全部验收标准；PRD §8 + `.claude/rules/e2e-testing.md`

### What to build

经 Workflow 派多纬度 subagent 做端到端验证，**主 agent 不自己点浏览器**，通过标准 = 浏览器可见 + DOM/computed-style 二确（非后端 artifact）。

### Acceptance criteria

- [ ] 经 Workflow 编排 ≥6 纬度 subagent，各自开自己的 tab、串/并行不抢资源
- [ ] 纬度：① 驱动方一致（agent CLI vs 人 UI，scenes[]+script.md 字节级一致）② 并发写无丢写 ③ 生成 handoff（注册+回链+stale+重拍，无悬挂引用）④ 既有 0.1.5 旧 work 迁移（引导态不崩不误导）⑤ 呈现-locale（zh/en×暗亮，无 raw 英文、徽章本地化）⑥ tab 不被弹走
- [ ] 每纬度截图证据 + DOM/computed-style 二确（视觉 finding 必二确）；completeness-critic subagent 汇总找漏测
- [ ] 任一纬度 UI 没动 → 如实标 fail，不靠后端 artifact 包装通过

### Blocked by

- S3, S4, S5, S7
