  - **(c) 主线（B7 与 B4 一体）**：`src/ws-bridge.ts` `buildSystemPrompt` 素材生成段瘦身为「端点名册」（capability + method + path，让 agent 知道有什么）+ 段尾稳定骨架句「所有素材生成端点参数以 `autoviral docs _shared/03-cli-reference` 为准——现查 docs，不要凭记忆里的旧参数」。**参数表全部移出提示词、只活在 manual**（B4 已迁齐 tts/captions/mix/transitions/frames）。B2 加的兜底禁令（不存在的 `.py` → `autoviral ask`）作为稳定纪律保留在提示词。`ws-bridge.test` 与 `docs-drift.test` 的提示词锁有意识改写：语义从「prompt 含参数表」改为「prompt 含名册 + docs 指针，manual 含参数表」，绝不为过测试留双份参数表。
  - **(a)-lite 辅线**：`src/ws-bridge.ts` 新增 `PROMPT_VERSION` 常量 + `PROMPT_CHANGELOG` map（version → 简短 *陈述句* context，措辞实证过 Opus 对 system 通道命令式 override 有抵抗，故全用陈述句不用命令式）。sidecar `SessionRecord` 加 `lastInjectedPromptVersion`（per-session，与 cliSessionId 同层，因 ADR-008 每 session 各有 CLI lineage 各自落后）。`spawnCli` 加 `--append-system-prompt` 形参；resume 分支经 `resumePromptAppend(workId, sid)` 计算「自 stored 版本以来的 changelog 拼接」并传入，成功后把 stored 推进到 `PROMPT_VERSION`；fresh 分支在 `system.init` 写回时直接记当前版本。**仅对 work-bound 会话生效**（coach/trends 跑别的提示词，helper 内 `isWorkBound` 守卫排除）。单测覆盖：落后→args 含 `--append-system-prompt` 且内容来自 changelog；持平→不含；版本推进正确；fresh 记版本；legacy 无版本→注全量。
- **未实测/风险**：`--append-system-prompt` 在 `--resume` 下把追加段插进既有 transcript 的**实际插入位置与生效性**未起真会话探针（侦察确认 flag 存在且无互斥，实现按文档行为做）；E2E 由后续 Workflow 多纬度验。
