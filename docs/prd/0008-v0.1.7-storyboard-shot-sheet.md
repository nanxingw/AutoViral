## 背景与问题

用户原话（2026-06-10）：「现在的剧本分镜给我的交互感不是很好，按钮很多很杂」。

多 agent 盘点（workflow `wf_69a6b9f3-384`，9 Opus subagent：盘点 + 4 角度提案 + 2 立场评审 + 合成）证实并量化了问题：

1. **控件密度失控**：每张 SceneCard 常驻 ~12 个裸控件（标题输入 / ↑↓ / intent 下拉 / prompt textarea / narration textarea / 时长 / 景别下拉 / 运镜下拉 / 生成按钮…），一屏 6 卡 ≈ 72 个可见控件，全部 data-bare 微型 mono 字号、无分组折叠。
2. **人侧增删断路**：UI 内**没有任何**「新建分镜」「删除分镜」控件——空状态只有一句"让 agent 用 `autoviral scene add`"的文案；`sceneEdit.ts` 仅有 patch/generate/reorder。server 端 `POST /scene` 与 `DELETE /scene/:id` 路由**已 wired**（bridge routes），纯前端缺口。
3. **状态不可读**：stale/generated 用 8px 纯色点区分，mono 小字号下几乎不可分（e2e Hard rule 5 颜色误读教训直接适用）。
4. **可发现性缺口**：mdAnchor 回链是死文案不可点；整卡拖拽重排零 affordance；inline-commit 离焦即存但无任何"已保存"反馈。

## 选定方案：A · 折叠镜表（双榜评审第一，85/86）

4 个角度提案（渐进披露 / 文档优先 / 看板优先 / agent 对话优先）经重度使用者 + 维护工程师两个立场独立打分，排序完全一致。选 A 的理由：降密度力度最锐（~72→~3 常驻 + 每行 1 个 hover ⋯）、零不变量风险、成本最低（纯前端、零新 endpoint）、可渐进交付。

**核心机制**：
- **折叠态镜头条**（默认）：每镜一行只读摘要——镜号 · 状态点+文字徽章 · 标题 · 时长 · 景别 · 意图 · hover ⋯ 菜单。已生成镜附 28px 缩略图。
- **选中态就地展开 Inspector**（手风琴，同时只 1 条展开）：全部字段编辑（title/prompt/narration/duration/shotSize/camera/intent）+ ↑↓ 重排 + stale 警示行 + 生成/重拍 CTA + ✓已存微反馈。
- **⋯ 菜单**：上移 / 下移 / 删除（两步确认，参照 #66 two-click-confirm，不抢全局 Delete 键）。
- **＋ 新建分镜**：列表底部常驻 + 空状态主按钮 → `addSceneRemote` POST /scene（与 agent `scene add` 同路径）。
- **状态三态多重编码**：◌ 空心=planned / ● 实心 accent=generated / ◍ 实心琥珀 + 「需重生」mono 文字徽章=stale（守 e2e Hard rule 5，不靠颜色单编码）。
- **剧本编辑器**：保留现有 edit/preview，外加 ▾ 折叠开关（默认展开），叙事层与执行层分层。

## 不变量（不可破坏，沿用 ADR-012 / CONTEXT 不变量 10/11）

- 所有写仍走 per-intent bridge 路由（patch/reorder/generate/add/remove），与 CLI 动词同一 shared scene ops——agent-人一致。
- `comp.scenes` 在 store 是只读镜像，UI 绝不本地 mutate，composition-changed → refetch 回流。
- 生成是下游 handoff：CTA 只在展开 Inspector 内、不建队列、产物不自动铺 timeline、in-flight 是瞬时 UI 态不入持久化。
- order 由 ops 拥有（reorder 传完整 permutation），UI 不写 order。
- 剧本/分镜弱链不变：mdAnchor 不做双向 diff、不假装对齐。
- inline-commit 行为保留：blur/Enter 提交、focus 中不被外部 refetch 顶掉草稿、title 必填、空值→null 清空协议、生成防重入。
- scenes 不进 800ms 整份 autosave。

## 范围外（诚实延后，非"以后再做"的借口——是评审点名的独立增量）

- grip 拖拽柄 affordance（保留现有整卡 drag-drop 路径不动）。
- mdAnchor 点击跳转导航（保留现状"未关联"文案逻辑）。
- 「展开全部」多镜并排比对逃生阀。
- animatic 预览（评审双榜点名：规划阶段 scenes 未上 timeline，"驱动主 Player"语义不成立，需 scene 级 composition 净新建——L 阶段另立项）。
- B 案的缩略图 filmstrip / 网格视图。

## 验收标准

1. 折叠态每卡**零常驻表单控件**（仅 hover/focus 显 ⋯）；展开态才出现全部编辑字段；同时最多 1 卡展开。
2. 人在 UI 能新建分镜（空状态 + 列表底部）和删除分镜（两步确认）；与 agent `autoviral scene add/remove` 产出字节级一致。
3. stale 镜显示「需重生」文字徽章 + 琥珀点；planned/generated/stale 三态不靠颜色即可区分。
4. 字段提交成功有 ✓ 微反馈；失败仍 role=alert 红字。
5. `npm run test:web` 全绿（新增折叠/展开/新建/删除确认/徽章断言）；`web typecheck` 0 错。
6. 多纬度浏览器 E2E（轻量 ≥3 纬度 Workflow）：人 UI 路径 / agent CLI 同步路径 / 边界（空状态、stale、删除确认取消）。
