## 0. 定位升级（北极星）与本版切片

用户对 AutoViral 的定位升级：从"简单短视频工具" → **万能视频生成器，支持电影级视频生成与编辑，全方位 AI-native**。这是跨多版本的北极星，必须分层：

| 层 | 北极星诉求 | 版本归属 |
|---|---|---|
| **L0 命题**（已存在不变量）| agent 驱动与人手动产出一致（ADR-009 共享 ops） | 已交付，本版复用 |
| **L1 剧本 · 分镜规划层** | "生成前先排剧本(md) + 素材区放剧本/分镜 + 一幕一幕生成" | **← v0.1.6 专做这一片（计划层 + 生成 handoff）** |
| **L2 生成执行精装修** | 异步生成 job 队列 / 多幕并发编排 / 取消 / 逐镜 i2v 尾帧连续性 / video 候选 | **v0.1.7+，明确 out-of-scope** |
| **L3 对齐剪映高级编辑** | 蒙版 / 抠像 / 曲线变速 / HSL 调色 / 降噪 / 防抖 / 花字 | **v0.1.7+ roadmap** |
| **L4 电影级多层叙事** | Act / Sequence 层级、跨幕导航、上百 scene | **v0.1.8+，schema 不预埋未实现层级字段** |

### 关键设计原则：计划与执行解耦（用户 2026-06-08 校准）

剧本/分镜是**计划层**（对齐 + 协作编辑），逐幕生成是**计划定好后的下游执行**。二者**解耦**：

- 计划层 = 本 PRD 的心脏（剧本 md + 分镜骨架卡片 + 双向编辑 + agent-人一致）。
- "生成此幕" = 一个 **handoff**：把分镜的 prompt 交给**现有生成流程**（现成的生成弹框/端点，原样复用），产物回素材库并回链到该分镜。**计划层不拥有生成引擎、不管它同步异步**。
- 因此**异步生成队列 / 并发编排 / 取消 / 实时生成驾驶舱 = L2，不在本版**。生成引擎的健壮性维持现状，单独 roadmap 升级。

## 1. Why / Problem Statement

剧本是视频创作的重中之重——决定整个创作方向，专业制作要花大量时间打磨。AutoViral 现状：

- **剧本只活在对话里**：agent 帮用户构思/编排剧本只能在 Chat 用文字来回，用户想改也只能再发一句话。交互差、浪费 token、没有一个"对齐总体流程"的可视面。
- **地基早就建好却睡着了**（built-not-wired，本仓库反复出现的病）：
  - `SceneSchema`（`src/shared/composition.ts:442`）字段完整、注释亲口说是"planning + dive-canvas affordance、order 独立于 timeline"——但**从没被写入、UI 零渲染、CLI 无 scene 动词、后端不消费**。
  - agent 创作时早就在 `plan/brief.md` / `plan/shotlist.json` 写东西，但 manual 亲口承认"the Studio doesn't render these"——**用户看不见**。
  - AutoViral 自己 `docs/archive/specs/2026-03-18` 早就定义过中文分镜表字段（编号/画面描述/首帧描述/时长/文案-旁白/镜头运动/尾帧描述）——**已有术语沉淀，应复用而非另造**。

**不做的代价**：最该被可视化、最该让用户介入的计划环节，继续埋在对话里；最大一笔 built-not-wired 资产继续睡着；电影级定位没有起点。

## 2. 目标用户（for whom）

- **创作者（人在 UI）**：想在生成视频前先排好剧本与分镜，像专业制作那样一眼看全片骨架、逐镜调整，而非靠和 AI 反复对话。
- **CLI agent**：能读用户剧本意图、据此排分镜，并与用户在 UI 的手动编辑**产出一致**。
- **进阶/电影级用户（未来）**：本版打的单层分镜地基，未来折叠展开即支持多幕长片，无需架构切换。

User story：
- *作为创作者*，我想在素材区有一块「剧本 · 分镜」面板，先写/让 agent 起草剧本总纲（可编辑 md），再看到逐镜分镜表，逐镜改画面/旁白/时长/景别/运镜，对某镜点「生成此幕」交给生成流程——全程可见、可手改、可让 agent 改。
- *作为 agent*，我想有 `autoviral scene`/`script` 动词，能据用户剧本排分镜，且我写的和用户在 UI 改的落到同一份 composition、永不漂移。

## 3. 术语：剧本 + 分镜脚本（两份联动产物）

专业影视/短视频创作里这是**两份不同文档**：

| 文档 | 专业名 | 是什么 | 类比 | 落点 |
|---|---|---|---|---|
| **剧本** | Script / Screenplay | **叙事层**：整体框架、故事梗概、旁白/台词、每段意图。自由文本。 | ≈ **PRD** | `plan/script.md` |
| **分镜（脚本）** | 分镜头脚本 / Storyboard | **执行层**：把剧本拆成一个个镜头，每镜带画面/景别/运镜/时长/旁白。结构化、逐行。 | ≈ **issue 切片** | `comp.scenes[]` |

- 对外功能名 = **「剧本 · 分镜」**；单个单元对外叫 **「分镜」**（一个分镜 = 一个镜头/一段）；**代码保留 `scene`**（已有 schema+测试，不改名，注释钉死 scene=一个分镜）。
- **故事板（storyboard）** = 分镜的画格版；**动态分镜（animatic）** = 故事板按时长串成的带配音粗剪——AutoViral 的 Remotion 预览**天然就是 animatic**（相对剪映的天然优势）。

**职责不重叠**：
- `plan/script.md` = **纯叙事层**：框架/梗概/旁白/每段意图。**不承载结构化逐镜清单**。
- `comp.scenes[]` = **结构化分镜表层**：逐镜的结构只在这里。
- 二者用 `scene.mdAnchor`（指向 md `## 段标题`）**弱关联、可漂移、不强一致**；UI 诚实呈现"独立维护、可能不同步"。

**层级模型（单层，不为电影级做架构切换）**：

```
Project 项目（= 一个 work / composition.yaml）
└─ [Act 幕 / Sequence 段落]   ← 电影级才展开，v0.1.6 不建 schema
   └─ Scene 分镜（= 升级版 SceneSchema，v0.1.6 一等公民 = 一个镜头/段）
      └─ Clip 片段（= 现有 VideoClip/AudioClip，落 timeline）
```

> **诚实简化**：专业上「场(scene)」按地点/时间分组、下含多个「镜头(shot)」。v0.1.6 单层，一个 `scene` 实际是**一个镜头/分镜**（短视频一行分镜表 = 一个 cut，本就这么用）。电影级(L4)再在上面加「场/段/幕」分组层，届时今天的单元下沉为 shot。**现在不预埋这层。** intent 保持 `hook|build|payoff|cta`（短视频语义），电影 vocab 延后。

## 4. 数据模型：升级 SceneSchema（桥接模型）

### 4.1 字段集（全部新增字段 optional/有 default，0.1.5 旧 work 原样 parse）

升级 `SceneSchema`（`composition.ts:442`）。**保留现有 `id/order/title/prompt/memberClipIds/memberAssetIds/intent` 全不动**，`order` 维持 required（不改既有契约）：

| 字段 | 中文 | 落点 | 档位 |
|---|---|---|---|
| `id` | —— | 现有 | 核心 |
| `order` | 镜号/序号 | 现有，**由 op 自动分配 `max+1`** | 核心 |
| `title` | 标题（一句话这镜干嘛）| 现有 | 核心 |
| `intent` | 叙事节拍（钩子/铺垫/爆点/CTA）| 现有 enum | 核心 |
| `prompt` | **画面描述**（直接可当生图/生视频 prompt）| 现有 | 核心 |
| `narration` | **旁白 · 文案**（口播/字幕台词）| **新增** `z.string().optional()` | 核心 |
| `durationSec` | 预计时长 | **新增** `z.number().optional()` | 核心 |
| `shotSize` | 景别（远/全/中/近/特写）| **新增** enum optional | 一等可选 |
| `cameraMovement` | 运镜（推/拉/摇/移/跟/固定）| **新增** enum optional | 一等可选 |
| `memberClipIds` / `memberAssetIds` | 关联片段 / 素材 | 现有 | 核心 |
| `generatedAssetIds` | 生成产物（回链）| **新增** `z.array(z.string()).default([])` | 核心 |
| `selectedAssetId` | 当前选中产物 | **新增** optional | 核心 |
| `status` | 状态：planned 待生成 / generated 已生成 / stale 需重生 | **新增** `z.enum([...]).default("planned")` | 核心 |
| `mdAnchor` | 关联剧本段落 | **新增** optional | 可选 |

> **stretch（不进本版 schema）**：`firstFrame`/`lastFrame`（i2v 尾帧连续性）—— 与 L2 生成精装修一起延后，**不预埋**（守"不预埋未实现字段"纪律）。

**规范（一份合格分镜的最低要求）**：每镜**至少有** `prompt`(画面描述) + `narration`(旁白) + `durationSec`(时长) + `intent`(意图)；薄了诚实标"待补"，不假装完整。

### 4.2 落地铁律（spec review 核出的真坑，必须遵守）

1. `CompositionWriteSchema` 是 `.strict()`，`order` required 无 default → **addScene 必须分配 order，否则写盘 zod 直接失败**（`composition-ops.ts:99`）。
2. **新建 `newSceneId()` 原语**（mirror `newTrackId`，`composition.ts:745`；仓库当前只有 `newTrackId`/`newClipId`）。
3. `addScene` op 内部 mint sceneId + 自动分配 `order = max(现有 order)+1`（空则 0），**调用方/CLI/bridge 不传 order**。
4. `reorderScenes` / `removeScene` 后强制重写 order 为连续 0..N-1（mirror `recompactDisplayOrder`，`track.ts:119`），加连号不变量单测。

### 4.3 持久化

- **scenes 内联 `Composition.scenes`**（L670 已 optional）→ 自动继承 `composition-changed` 刷新链（`useBridgeEvents.ts:128-144`）。单一 work SSoT。
- `plan/script.md` 是独立文件 → 单独 `plan-changed` 事件 + watcher。
- 性能 open：电影级上百 scene 内联会让整份 refetch 变重；v0.1.6 单层短视频 scene 少，暂存 composition.yaml；L4 若爆迁独立 `script.yaml`（可逆）。

### 4.4 并发写：诚实降级为轻量加固（spec review 纠正）

> **spec review 纠正**：`mutateCompositionFor`（`composition-ops.ts:258-283`）是**无锁 read-modify-write**——两个并发 per-intent 写都读到同一份 `current`，后写者覆盖前写者，lost update 存在。**这不是剧本层引入的**，现有 11+ 动词早就如此（罕见碰巧同改）。剧本层是**计划编辑**（低频），并发频率与现有动词同级，**不是被生成洪流猛冲的并发常态**（因为生成已解耦为 handoff）。

- **scenes 走 per-intent 即时写**（与 agent 同路径），**不进 Studio.tsx:186 的 800ms 整份 autosave**（autosave 用陈旧整份快照覆盖的窗口最大，scenes 绕开它）。
- **轻量加固（推荐 slice，非命题攸关）**：给 `mutateCompositionFor` 加 **per-work 写串行化（async mutex）**，根治所有动词（含 agent 整份 `/comp` PUT `routes.ts:203`）的 read-modify-write lost-update。**不夸大为"结构性消灭"**——它是把竞态窗口收敛到接近零的正确加固，顺带修既有动词。若本版时间紧，可标 stretch；但因 ROI 高（一处修全部动词）建议进。

### 4.5 事实源职责切分（写进 CONTEXT.md 不变量）

- `plan/script.md` = **叙事意图**事实源（自由文本，纯叙事层）
- `comp.scenes[]` = **机读分镜表** SSoT（经 strict zod chokepoint 写）
- 二者 `sceneId ↔ mdAnchor` **弱关联、可漂移**；不自动双向 diff（见 §6）

## 5. 逐幕生成 = 下游 handoff（不内嵌驾驶舱、不建异步队列）

> **设计原则**：计划层定好分镜后，「生成此幕」只是把这一镜**交给现有生成流程**，不在剧本面板里重建生成机器。

### 5.1 触发与产物注册（复用现有路径，修悬挂引用）

- 卡片「生成此幕」→ 用该分镜的 `prompt`（可拼接 `shotSize`/`cameraMovement`/`narration` 富化）调**现有生成流程**（`GenerationDialog` 预填 ∥ 现有 `/api/generate/*` 端点）。
- **产物必须注册为 `AssetEntry`（image/video 路径不对称，spec review 核出）**：video 经 `/api/providers/:id/generate-video` 会写 `AssetEntry` + `ProvenanceEdge`（`generate.ts:548-574`）；但 **image `/api/generate/image`（`generate.ts:37-83`）只广播 `asset-added`、不写 `composition.assets`**。因此 handoff 回链前**必须确保产物已登记为 `AssetEntry`**——image 路径需补登记（或由 `linkSceneAssets` 兜底建条目），否则 `generatedAssetIds` 指向 `composition.assets` 里不存在的 id = 悬挂引用。P5 必须把"登记 AssetEntry → 再 linkSceneAssets"做成单一可靠链路。
- **回链到分镜**：生成成功后据 HTTP 响应的 `assetPath` → `linkSceneAssets({sceneId, assetIds})` 写入 `generatedAssetIds` + `selectedAssetId`，`status=generated`。归属靠**请求-响应一一对应**（`asset-added` 广播不带 sceneId，`generate.ts:72-77`，不用它做归属）。

### 5.2 状态（轻量，持久在 scene 经 ops 写）

- `planned` 待生成 → `generated` 已生成（有产物）→ 改 `prompt`/输入后 → `stale` 需重生（脏标记，**只标被改的这一镜**，不做下游传播）。
- **生成中/失败是瞬时 UI 态**，由现有生成流程自己的进度/错误 UI 负责（剧本面板不持久化 generating/failed，不重建状态机）。
- agent 与 UI 看到同一份 `status`/`generatedAssetIds`（经 ops 写进 composition.yaml）→ 守 agent-人一致。

### 5.3 重拍 / 候选（轻量）

- 单镜「重拍」= 再走一次 handoff，新产物 **append 到 `generatedAssetIds`**，`selectedAssetId` 指向当前；卡片可切换历史 take。
- **多候选一次出 N 张**：image 复用现成 `/api/generate/image/batch`+`/api/frames/select`（key 由 `shotId` 换 `sceneId`，spec review 核实 re-key trivial）。**video 无候选基础设施**（batch 仅 image），video 多候选 = **L2 stretch，本版不做**。

### 5.4 不落时间线

产物只回填 `scene.generatedAssetIds`/`selectedAssetId`，**不自动铺 timeline**（用户决策）。卡片不放"上时间线"按钮，避免覆盖已精剪结果（规划层与 timeline 解耦）。

### 5.5 明确不做（L2，本版 out-of-scope）

异步生成 job 队列 / 多幕并发编排 / 真取消（Seedance provider 内部 poll 最长 ~5min 无 AbortController，`seedance.ts:59-60`，真异步化是 L2 子项目）/ video 候选 / i2v 尾帧→首帧连续性 / 实时 per-scene 生成驾驶舱。生成引擎健壮性维持现状。

## 6. 双路径编辑：保证两端产出一致

**不变量：scenes 所有 mutation 永远经同一份 `@shared/composition/ops` + `mutateCompositionFor` chokepoint（ADR-009），两端都走即时 per-intent 写。**

### 路径 A：用户直接编辑
- 改 md：ScriptPanel textarea → `PUT /api/works/:id/plan/script.md` → broadcast `plan-changed` → refetchScript。
- 改分镜卡片：inline 编辑/拖拽 reorder → **直接 per-intent scene 路由**（不进整份 autosave）→ 同一份 ops → `composition-changed` → 回灌 store。

### 路径 B：agent 按指令改
- 改 md：`autoviral script edit` → writeFile → `plan-changed` → 面板实时更新。
- 改分镜：`autoviral scene set <id> --intent payoff` → bridge → 同一份 ops → `composition-changed`。

### 一致性如何保证
1. **scenes 双端共消费同一 ops**（mirror splitClip，`routes.ts:1135` 注释："EXACT implementation the studio store runs"）。
2. **scenes 两端走同一条即时写路径**（§4.4），加 §4.4 的 per-work 写锁后竞态窗口收敛到接近零。
3. **md 只有一个事实源** `plan/script.md`，两条路径写同一文件。
4. **md ↔ scenes 不自动双向 diff**：用户改 md 结构不由后端自动解析回 scenes；同步由 agent 显式跑 scene 动词。`mdAnchor` 由调用方/agent 显式传，**没传则空、卡片显"未关联剧本段落"**——不猜、不假装对齐。

## 7. UI 设计：素材区第三个 tab「剧本 · 分镜」

**放哪**：右栏 AssetSidebar（~17% 宽）加第三个 tab。短视频 scene 少，17% 够；电影级独立第四视图延后 L4。

> **spec review 修**：AssetSidebar 现有 `if (selection) setTab('inspector')`（选中任一 clip 自动跳 inspector tab）会把用户**弹出剧本 tab**。修：用户停在剧本 tab 时**不自动跳走**（剧本 tab "粘"住，或 auto-switch 仅在用户未手动停留剧本 tab 时生效）。

**长什么样**（editorial·cool·glass）：
- 上半 **剧本 md 编辑器**（编辑 ↔ react-markdown 预览切换）+ 顶部"剧本与分镜独立维护、可能不同步"诚实提示。
- 下半 **分镜脚本卡片列表**（读 `comp.scenes`，按 order），每镜：镜号 + 标题 + intent 徽章 + status 点（待生成空心/已生成实心+缩略图/需重生）+ 画面描述摘要 + 旁白 + 时长 + 景别/运镜（可选字段，有才显）+ 「生成此幕」「重拍」+ `mdAnchor` 缺失显"未关联剧本段落"。

**既有 work（无 scenes 无 script.md）引导态**：显本地化"还没有剧本，写一份或让 agent 起草"+「新建剧本框架」按钮；**不空白不报错、不从已有时间线反推造假关联**。`hasScript` 与 `hasScenes` 两个独立信号分别判定，不互相暗示（spec review）。

**诚实薄数据守门**：
1. 空 md **不硬编码任何语言**：GET 404 返回空字符串，前端显本地化 placeholder（避免 #73/#83 式 i18n-string-as-data bug）。
2. md↔scenes 漂移诚实呈现（顶部提示 + 缺锚点显"未关联"）。
3. status 如实；薄字段标"待补"。

**全部新文案 + aria-label 进 `messages.ts` en/zh 双份**，加测试断言 EN-locale 渲染无中文字符。

## 8. 验收标准（E2E 以浏览器可见为准）

> 遵守 `.claude/rules/e2e-testing.md`：通过唯一标准是**浏览器里看得到 + DOM/computed-style 二确**，不是后端 artifact。E2E **经 Workflow 派多纬度 subagent**，主 agent 不自己点浏览器。

1. **剧本 md 可见可编辑**：素材区「剧本·分镜」tab 打开，能写/编辑 `plan/script.md`，编辑→预览切换正常；刷新后持久（DOM `textContent` 二确）。
2. **分镜脚本可见可改**：`comp.scenes` 渲染成卡片（按 order）；inline 改 title/画面描述/旁白/时长/intent/景别/运镜 + 拖拽 reorder 后刷新可见、落盘持久。
3. **agent-人一致**：同一意图分别走"agent 经 CLI"和"人在 ScriptPanel"，对比 `composition.yaml` scenes[] + `plan/script.md` **字节级一致**（浏览器可见 + DOM 二确）。
4. **并发写加固有效**：agent `POST /scene` 与 UI 卡片编辑交错，断言两端写都落盘（验 per-work 写锁；若锁标 stretch 则降级为"低频下不丢"并诚实说明）。
5. **生成 handoff**：点「生成此幕」→ 走现有生成流程 → 产物进素材库（**composition.assets/provenance 有此 asset，无悬挂引用**）+ 回链该分镜、卡片显缩略图与已生成；改画面描述 → 卡片转 stale"需重生"。
6. **既有 0.1.5 旧 work 迁移**：真旧 work 打开「剧本·分镜」tab 显引导态、不崩、不误导。
7. **呈现/locale 纬度**：zh/en × 暗亮主题各截面板，DOM 二确无 raw 英文、status/景别/运镜徽章本地化。
8. **tab 不被弹走**：停在剧本 tab 时点选 timeline clip，不被自动切到 inspector（DOM 二确当前 tab）。

## 11. 风险 + 回退

| 风险 | 影响 | 回退/缓解 |
|---|---|---|
| addScene 缺 order 第一次写盘 zod 失败 | P0 落盘即炸 | order 由 op `max+1` 分配；reorder/remove 连号化；P0 显式断言 |
| 并发写 lost-update（既有 + scene）| agent/UI 写互相覆盖 | P0.5 per-work 写锁根治（顺带修既有动词）；scenes 不进整份 autosave |
| 空模板硬编码语言 = i18n bug | 重演 #73/#83 | 404 返回空串，本地化 placeholder，无硬编码模板 |
| 生成产物悬挂引用（scene 有 id 但 assets 无）| 卡片引用不到素材 | image 路径不写 composition.assets（仅 video 写）→ P5 统一"先登记 AssetEntry 再 linkSceneAssets"，覆盖 image |
| AssetSidebar 选中 clip 把用户弹出剧本 tab | 编剧本时被打断 | 剧本 tab 粘住，auto-switch 不覆盖手动停留 |
| md↔scenes 漂移无人对账 | UI 假装强关联 | 不自动同步；UI 诚实标注 + mdAnchor 缺失显"未关联" |
| 既有 work 打开 tab 空白吓到用户 | 误以为数据丢失 | 本地化引导态，不空白不报错、不反推造假 |
| 生成引擎本身慢（video ~5min）| 用户等待 | **本版不解决**（L2）；生成走现有流程现状体验；剧本层只 handoff 不背锅 |
| agent 不会用新能力 | scene/script 动词成摆设 | P3 同步 buildSystemPrompt + manual + recipe；E2E 用真 agent 验 |
