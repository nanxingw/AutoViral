## Phase A — 止血 + 地基契约（无依赖，可立即并行）

### S1 · 止谎：修掉文档/注释/CLI help 的假承诺
**类型**：AFK · **覆盖 US**：35,36,37

**What to build**：清掉调研复核坐实的"文档撒谎"——agent 高度信任手册，照做必败会烧掉整个会话预算。把官方 crossfade recipe 改成真能跑通的路径（走 `comp put` 或 keyframe 动词，不再给出必报 400 的 `clip set --keyframes`）；CLI help 不再列运行时会 throw 的 `overlay` track（或在 S10 真支持后保留）；删除源码里指向不存在 UI 的假注释（"caption import button calls this"）；变量变速 export 若静默回 1× 则发 `warn` 给 agent。

**Acceptance criteria**：
- [ ] 按官方 crossfade recipe 逐字执行每条命令，全部返回成功、最终转场真实出现（经 workflow E2E 验证）。
- [ ] `autoviral clip add --help` / CLI help 不再宣称当前会 throw 的能力。
- [ ] 全仓 grep 不再有指向已删脚本 / 不存在 UI 的假注释（列出清单逐条删或改）。
- [ ] 变量变速且 export 实际回 1× 时，响应/日志带明确 warn。

**Blocked by**：None — can start immediately

### S2 · 写路径 broadcast：改完即刷新

**Acceptance criteria**：

**Blocked by**：None — can start immediately

### S3 · 错误码契约两端打通
**类型**：AFK · **覆盖 US**：18,19

**What to build**：让 agent 能按退出码做控制流分支。所有 400 响应一律带 `code:4`（目前 `/clip` 三端点漏带、只有 carousel 带）；`client.ts` 失败分支 `process.exit(json.code ?? 3)`；固定 4xx→exit 4（输入/校验错）、5xx→exit 3（服务错）、`ask` timeout→124 不变。

**Acceptance criteria**：
- [ ] 非法 clip 改动 → HTTP 400 `code:4` → CLI 退出码 4；服务错 → 5xx → 退出码 3。
- [ ] `error-codes.md` 契约与实际响应/退出码一致（按文档断言测）。
- [ ] CLI 集成测覆盖 4 vs 3 的分支。

**Blocked by**：None — can start immediately

**Acceptance criteria**：

- [ ] 回写后浏览器即反映（经 workflow E2E）。
- [ ] CLI 走的是 bridge 前缀，不存在硬编码绕过。

**Blocked by**：S2
