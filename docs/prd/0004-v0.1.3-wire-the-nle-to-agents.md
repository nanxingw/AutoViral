## Problem Statement

站在用户（这里"用户"= 驱动工位的 CLI agent，以及在浏览器里直接编辑的人）的角度：

- **作为驱动剪辑的 agent**，我以为我能驱动这台工位，但我实际只能 `clip add/set/remove` + `carousel add-slide/set-layer` + `ui-*` + `export/snapshot`。我想做分割、裁剪、跨轨移动、加转场、加关键帧、加轨道——**没有任何对应命令**，只能手搓整份合法 composition JSON 再回写，等于在 prompt 里重写前端 store 的 25 个 action 和它们的全部 invariant。
- **更糟的是我以为成功了其实没有**：`clip set vc1 --scale 2` 返回 `ok:true`，但嵌套字段被 JSON-flat 解析器 + 浅合并 + zod 默认 strip 静默丢弃，画面纹丝不动（silent no-op）。我自信交付了一个根本没改的作品。
- **官方 recipe 在教我跑必败命令**：`crossfade-between-clips.md` 自己写"schema expects an array"，紧接着却给出 `clip set --keyframes '<json>'`——必报 400，烧掉我整个会话预算。
- **改完前端不刷新**：5 个 composition/carousel 写端点没有一个广播 `composition-changed`，全靠脆弱的 `fs.watch`。我收到 HTTP 200 以为成功，用户却"什么都没看到"。
- **作为在浏览器里编辑的人**，剪映/OpenShot 都有的基本盘——crop、fit-fill 填充模式（素材比例≠画布时不被强制裁切）、画布比例一键切换、翻转镜像、倒放、定格——AutoViral 一个都没有；我改错一个 clip 还无法 Cmd+Z 撤销。
- **作为开发者**，意图级编辑逻辑只活在前端 `store.ts`，后端 bridge 够不着，任何想接通的尝试都会变成"前后端两套实现各自漂移"。

**一句话**：AutoViral 在浏览器里已经是一台真正可用的多轨 NLE（split/trim/ripple/跨轨/转场/关键帧/调色/混音全是真实现，preview 与 export 同走 Remotion 故 WYSIWYG），但它的核心命题「任意 CLI agent 经 `autoviral` CLI 驱动剪辑」在**写路径**上只覆盖一小块——富功能几乎全是 *built-but-human-only*，agent 够不着。

## Solution

v0.1.3 的最高杠杆**不是加新功能，而是把已经建好的浏览器 NLE 接通到 agent，并修掉信任崩塌的几处静默失败/文档撒谎**。具体：

1. **把意图级编辑逻辑下沉到 `@shared` 共享核心**（M1 keystone）——一份纯 mutation 实现，前端 store（immer draft）和后端 bridge（parsed object）共消费，彻底消除双实现漂移。
2. **沿 bridge chokepoint 补齐意图级动词**（split/move/transition/track/comp put/comp validate）+ 对应 CLI 动词，让 agent 不再手搓整份 JSON。
3. **修信任三连**：写路径显式广播 `composition-changed`（改完即刷新）、`clip set` 支持嵌套字段且拒绝静默 strip、错误码契约两端打通（agent 可按 `$?` 分支决策）。
4. **接通已建未接的最后一公里**：ASR 自动字幕落轨、平台 preset 真生效。
5. **止谎**：修掉 crossfade recipe、删 CLI help 的 overlay 假承诺、删源码假注释。
6. **补基础画面操作基本盘**：crop / fit-fill / 画布比例切换 / 翻转镜像 / 倒放 / 定格（schema + Remotion 渲染消费 + M1 ops + CLI）。
7. **给人和 agent 都装上安全网**：clip 级 undo + Cmd+Z、agent 可达的 checkpoint restore（restore 前先建当前态）。

护城河（auto-reframe 复活、AI 自动化套件、真调色+抠像、carousel 服务端合成、长任务 job 模型、人机并发守卫）划入 **v0.1.4 独立 PRD**，本版不做（见 Out of Scope）。

## User Stories

### Agent 写路径 · 意图级动词（M1/M5）

1. 作为 agent，我想用 `autoviral clip split <id> --at <sec>`，以便在指定时刻切开一个片段而不必手算两段的 in/out/offset 和 keyframe 重基。
2. 作为 agent，我想用 `autoviral clip trim <id> --in <sec> --out <sec>`，以便收窄/扩展片段并自动遵守邻接 cap、最小时长、keyframe 重基。
3. 作为 agent，我想用 `autoviral clip move <id> --to-track <trackId>`，以便把片段移到另一条同类轨（如把配音从 A1 挪到 A2），保留 trackOffset。
4. 作为 agent，我想用 `autoviral transition add --track <id> --after <clipId> --preset <name> --duration <sec>`，以便在两个片段间加一个时间线转场。
5. 作为 agent，我想用 `autoviral transition remove --track <id> --id <transitionId>`，以便移除一个转场。
6. 作为 agent，我想用 `autoviral track add --kind <video|audio|text|overlay>` 和 `autoviral track remove <id>`，以便新建/删除轨道（如先建 A2 再往里放配音）。
7. 作为 agent，我想用 `autoviral clip add` 能往**具体某条轨**（按 trackId）添加，而不是只命中第一条同 kind 轨，以便区分 A1·BGM 与 A2·VO。
8. 作为 agent，我想 `autoviral clip add --track overlay` 真的能加 overlay 片段（贴纸/PiP），而不是运行时被 throw"not yet supported"。
9. 作为 agent，当我做以上任何意图级操作时，我想要它们遵守与前端 UI **完全一致**的语义和 invariant（因为它们是同一份 `@shared` 实现），以便我在浏览器里看到的结果和我命令产生的结果一致。

### Agent 写路径 · 整份回写 + 预检（M3/M5）

10. 作为 agent，我想用 `autoviral comp put <file|-stdin>` 把一份完整 composition 回写，作为任何富改动的万能逃生口——而且这条命令要走 bridge 前缀，不必我自己拼 `/api/works` raw curl。
11. 作为 agent，我想用 `autoviral comp validate <file|-stdin>` 在落盘前校验一份候选 composition（zod + lint + inspect），拿到结构化 issues，以便把"PUT→400→读 zod dump→猜"的昂贵循环砍掉。
12. 作为 agent，我想给任何写端点加 `--dry-run`，以便预演一次改动看它是否合法/会触发什么 lint 警告，而不污染用户正在看的 composition。

### Agent 写路径 · clip set 嵌套字段（M2/M7）

13. 作为 agent，我想 `autoviral clip set <id> --scale 2` 真的把 `transforms.scale` 改成 2，而不是返回 `ok:true` 却被静默 strip。
14. 作为 agent，我想能改任意嵌套属性（`transforms.x/y/rotation/opacity`、`filters.brightness/contrast/saturation`、文字 `style.color/fontSize/...`、`fade`、`ducking`），以便真正驱动"agent 改图"。
15. 作为 agent，当我拼错一个 key（`--scal 2`）或传了一个该 kind 不支持的字段时，我想得到一个 **400 报错**而不是被静默吞掉，以便我立刻知道哪里错了。
16. 作为 agent，我想用 `autoviral clip keyframe add/set` 显式增改关键帧（数组字段），以便 crossfade / Ken Burns 这类需要 keyframe 的效果不再因为 `--keyframes` flag 必败而做不了。

### Agent 反馈环 · 信任（M4/M6）

17. 作为 agent，当我的任何写操作成功落盘后，我想前端立即刷新（preview 显示我刚做的改动），以便我能 snapshot 自检、用户也能立刻看到——而不是收到 200 但界面纹丝不动。
18. 作为 agent，当我的命令失败时，我想 CLI 退出码反映失败类别（4xx→exit 4 输入/校验错，5xx→exit 3 服务错），以便我能按 `$?` 做控制流分支（"我 patch 不合法"还是"服务挂了"）。
19. 作为 agent，我想所有 400 响应都带 `code:4`（目前 `/clip` 三端点漏带、只有 carousel 带了），以便错误码契约对所有写端点一致。

### 自动字幕（M8）

20. 作为 agent，我想用 `autoviral captions generate [--language <lang>]` 对当前作品的音频跑 ASR，并把带时间码的 segments 自动写进一条 text track，以便"自动配字幕"这个最高频诉求一条命令完成。
21. 作为在浏览器里编辑的人，我想要一个"生成字幕"按钮触发同一条流程，以便我不开终端也能用上后端早就完整的 ASR 能力。

### 平台 preset 真生效（M9）

22. 作为 agent，我想 `autoviral export --preset <抖音|小红书|微信|youtube|...>` 真的应用该平台的尺寸/响度（LUFS）/码率，而不是被静默忽略恒走 -14 默认。
23. 作为 agent，当我传了一个未知 preset 名时，我想得到 400 报错而不是被静默吞掉，以便我知道这个 preset 不存在。
24. 作为开发者，我想平台预设表是 `@shared` 单一事实源，前端组件和渲染管线读同一份，以便不再"前端硬编码、agent 拿不到"。

### 基础画面操作（M11）

25. 作为编辑者/agent，我想给视频片段设 fit-fill 填充模式（cover / contain-letterbox / blur-bg），以便素材比例≠画布比例时不被强制裁切。
26. 作为编辑者/agent，我想一键切换画布比例（9:16 ↔ 1:1 ↔ 16:9），以便同一份素材产出不同平台规格而不必手改 comp 数字。
27. 作为编辑者/agent，我想 crop（显式裁剪画面区域），以便"裁掉画面左 1/3"可表达。
28. 作为编辑者/agent，我想翻转/镜像片段（水平/垂直），以便纠正镜像素材或做对称效果。
29. 作为编辑者/agent，我想倒放片段（reverse），以便做倒放效果。
30. 作为编辑者/agent，我想定格某一帧（freeze-frame），以便做停顿强调。
31. 作为 agent，以上每种操作我都想有对应 CLI 动词，且它写进 schema 后**渲染器真的消费**（不重蹈 LUT 那种"写进 schema 却零渲染消费"的死字段）。

### 安全网 · undo / restore（M12）

32. 作为在浏览器里编辑的人，我想 Cmd+Z 撤销 clip 级操作（不只是 track 级），以便改错一个片段能退回。
33. 作为 agent，我想用 `autoviral checkpoint list` 看历史检查点、`autoviral checkpoint restore <id>` 回滚，以便我手搓 JSON 出错后能安全退回。
34. 作为系统，当 agent 触发 restore 时，我想**先把当前态建成一个 checkpoint** 再覆盖，以便 restore 本身不造成数据丢失（#68 教训）。

### 止谎（M10）

35. 作为 agent，我想官方 recipe 里的每条命令都真能跑通（crossfade recipe 改走 `comp put` 或 keyframe 动词），以便我信任手册。
36. 作为 agent，我想 CLI help 不列我加不了的 track 类型（删 overlay 假承诺，或在 M5 真支持后保留），以便 help 不撒谎。
37. 作为开发者，我想删掉源码里"caption import button calls this"这类指向不存在 UI 的假注释，以便注释不把 built-not-wired 说成 wired。

## Implementation Decisions

### 模块分解（与作者核对定稿）

**🔑 Keystone — 深模块（纯函数 / 可独立测 / 接口稳定）**

- **M1 `@shared/composition/ops` — 意图级 mutation 核心**。把 `store.ts` 现有 11 个意图 action（`addClip / updateClip / removeClip / splitClip / trimClip / moveClipToTrack / addTransition / updateTransition / removeTransition / addTrack / removeTrack`）提取成纯 mutation 函数，**原地修改一个 `Composition` 对象**——这对前端 immer draft 和后端 bridge 的 parsed object 都成立（immer draft 是可变 proxy）。封装全部 invariant：`trk_`/clip id mint、transition `afterClipId` 非末位约束、split/trim 的 keyframe 重基、speed clamp（0.1–4.0）、邻接 cap、跨轨同 kind 守卫、移动后源轨 transition 孤儿 prune。函数签名形如 `splitClip(comp, { clipId, atSec }): { newClipId }`（mutate + 返回 mint 出的 id），非法参数 throw 一个**带 `code` 的 typed error**。
  - store 侧：`set(s => ops.splitClip(s.comp, params))`。
  - bridge 侧：`mutateCompositionFor(ctx, comp => { ops.splitClip(comp, params); return comp })`。
  - 这是 **ADR-009** 的核心决策：意图级 mutation 单一实现，消除双实现漂移。

- **M2 `@shared/composition/patch` — deep-merge + per-kind 字段白名单**。`applyClipPatch(clip, patch): clip`（或 throw typed error）。解析嵌套路径（`transforms.scale`、`filters.brightness`、`style.color`、`fade.in`、`ducking.ratio`、`keyframes[]`），按 clip kind 的允许字段集校验每个 key，**未知 key 抛错而非静默 strip**。被 M1 的 `setClipProps` 和 bridge PATCH 路由复用。

- **M3 `@shared/composition/preflight` — 候选 composition 校验**。纯 `preflight(candidateComp): { ok, errors, warnings }`：`CompositionSchema.safeParse` + 复用 lint/inspect 规则，作用于**内存中的候选对象**（区别于现有 `src/composition/quality/*` 读 on-disk）。撑起 `comp validate` 与 `--dry-run`。

**🔌 浅适配层（薄管道，套在深模块上）**

- **M4 写路径事件契约**。修改 `mutateCompositionFor` / `mutateCarouselFor`（唯一写 chokepoint）：每次成功落盘后 `broadcast(workId, 'composition-changed' | 'carousel-changed')`。**一处修复覆盖全部 5 个写端点**。`fs.watch` 降为兜底。
- **M5 意图级 bridge 路由 + CLI 动词**。新 bridge 路由：`POST /split`、`POST /clip/:id/move`、`POST /transition` + `DELETE /transition/:id`、`POST /track` + `DELETE /track/:id`、`PUT /comp`、`POST /comp/validate`；`POST /clip` 支持按 `trackId` 定位并去掉 overlay 硬拒绝；`PATCH /clip/:id` 改走 M2。CLI 动词一一映射。`PUT /comp` 放在 bridge 前缀下 → **不需要 hack `client.ts` 的 base-URL**。
- **M6 错误码契约**。`client.ts` 失败分支 `process.exit(json.code ?? 3)`；固定 4xx→`code:4`、5xx→`code:3` 映射；给 `/clip` 三端点 400 补 `code:4`。
- **M7 CLI 嵌套参数解析**。`clip set` 对已知嵌套 key 做 JSON.parse（失败再退字符串）；新增 `clip keyframe add/set` 动词。
- **M8 ASR 字幕接通**。bridge `POST /captions/generate`（调已有 `/api/audio/captions`，把 segments 经 M1.addClip 写进 text track）+ CLI `captions generate` + Studio"生成字幕"按钮。
- **M9 preset 真生效**。平台预设表移 `@shared`；`runRenderPipeline` 接收并应用 `/export` body 的 `preset`（尺寸 + LUFS + 码率）；未知 preset → 400。
- **M10 文档/注释止谎**。crossfade recipe 改走 `comp put` / keyframe 动词；CLI help 与 M5 的真实能力对齐；删 `audio.ts` 假注释；变量变速 export 时若静默回 1× 则 `warn` 给 agent。
- **M11 基础画面操作**。`TransformsSchema` 扩 `crop`/`flip`；新 `fitMode`（cover|contain|blur-bg）；comp 级"画布比例切换" op；clip 级 `reverse`/`freeze`。**Remotion 渲染器（preview）+ ffmpeg post（export）双消费**（ADR-002 WYSIWYG）；M1 加对应 ops；CLI 加动词。
- **M12 安全网**。bridge `GET /checkpoints` + `POST /restore`（restore 前先 createCheckpoint 当前态，#68）+ CLI `checkpoint list/restore`；store clip 级 undo + `useShortcuts` 加 Cmd+Z。

### 关键契约 / Schema 变更

- `TransformsSchema` 新增（M11）：`crop?: {x,y,w,h}`、`flipH?: boolean`、`flipV?: boolean`、`fitMode?: "cover"|"contain"|"blur-bg"`；视频片段 `reverse?: boolean`、`freezeAtSec?: number`。新增字段**全部 optional + 有 default**，老作品 round-trip 不受影响。
- 平台预设（M9）下沉 `@shared`：`{ id, label, width, height, fps, loudnessTargetLufs, videoBitrate }[]`，前端 `PlatformPresetSection` 与 `runRenderPipeline` 读同一份。
- 错误码（M6）：4xx 一律 `code:4`，5xx 一律 `code:3`，`ask` timeout 仍 `code:124`（不变）。
- M1 ops 的 typed error 形如 `class CompositionOpError extends Error { code: number }`，bridge 把 `.code` 透传进 JSON 响应。

### 实现顺序（供 to-issues 切 tracer-bullet 纵切片）

止谎/止血优先（成本最低、信任回报最高）→ 再接命题写路径 → 再补功能基本盘：
1. **M10 止谎** + **M4 broadcast**（独立、立即止血）。
2. **M1 ops 提取**（keystone，先建 ADR-009；store 切换为调 ops 且**现有测试须全绿**，证明零行为变化）。
3. **M2 patch + M6 错误码** + **M7 CLI 嵌套解析**（修 silent no-op，端到端一条 `clip set --scale` 真生效）。
4. **M5 意图动词 + M3 preflight**（`comp put`/`comp validate` 先行作逃生口，再逐个补 split/move/transition/track）。
5. **M8 ASR** + **M9 preset**（接通已建未接的最后一公里）。
6. **M11 画面操作**（schema→render→ops→CLI 全栈，每个操作一个纵切片）。
7. **M12 undo/restore**（安全网）。

## Testing Decisions

**什么是好测试**：只测外部可观测行为，不测实现细节。对纯模块测"输入 comp + 参数 → 输出 comp / 抛错"；对协议层测"命令/HTTP → 响应 + 落盘后的 comp 状态 + 退出码"；对渲染测"schema round-trip + 渲染产物可观测属性"。**避免**断言内部函数被调用次数这类脆弱测试。

**要写测试的模块（作者勾选）**：

- **M1 `@shared/composition/ops`**（最高 ROI）：每个意图 op 的 happy path + 全部 invariant 边界——split 在片段边界/外部/keyframe 重基、trim 越邻接 cap clamp、move 跨 kind 拒绝 + 源轨 transition 孤儿 prune、transition 加在末位被拒、addTrack id 格式、speed clamp。**一份实现喂两个消费方**，所以这层测稳了，store 和 bridge 同时受益。
- **M2 patch + M3 preflight**：patch 的嵌套路径解析、未知 key 拒绝（最关键，防 silent no-op 回归）、per-kind 白名单；preflight 对合法/非法/带 lint 警告的候选 comp 返回正确 `{ok,errors,warnings}`。
- **M5/M6 bridge 路由 + CLI**：每个意图动词的 HTTP 集成测（请求→200/400→落盘 comp 断言）+ 错误码契约（400→`code:4`→CLI exit 4；500→exit 3）+ overlay 现在可加 + 按 trackId 定位。
