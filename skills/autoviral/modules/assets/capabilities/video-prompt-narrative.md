# Video Prompt — Narrative Layer

`dreamina-mastery.md` 是**工具书**——告诉你 `text2video` / `image2video` / `multimodal2video` 的命令矩阵、模型选型、运镜词汇。这份是**创作书**——告诉你**怎么把一个情感意图变成 prompt 字符串**。

两者关系：

```
情感意图  ──► 视觉决策（taste/02 的镜头语法）
   │              │
   │              ▼
   └────► video-prompt-narrative.md  ──► dreamina-mastery.md  ──► CLI 命令
                  （你在这）              （工具说明）
```

如果跳过这一层直接进 dreamina-mastery，prompt 会变成"镜头跟随主体走路、温暖光线、慢动作"——这是 AI 默认值的**语言外壳**，不是创作。

---

## 第一原则：每个 prompt 必须能回答 3 个问题

写之前问自己：

1. **观众这 8 秒之内想感受到什么？**（一个词：好奇 / 紧迫 / 敬畏 / 温暖 / 心痛 / 荒诞…）
2. **如果我去掉镜头运动，这个画面还成立吗？**（如果是，运动是装饰；不是，运动承载情感）
3. **第 1 帧和第 8 帧分别钉死了什么？**（想象停在这两帧——它们是不是各自完整的画面？）

**任意一题答不上**，回 `taste/` 重想，不要写 prompt。

---

## 8 秒微型叙事弧

短视频里**每个 clip** 都有自己的微型弧。一段 8 秒视频不是"一个连续画面"，是 4 拍：

```
0-1s  钩子拍 (Hook)        — 一帧抓住注意
1-4s  语境拍 (Context)     — 让观众确认看到了什么
4-7s  升级拍 (Escalation)  — 局势变化、张力提升、揭示
7-8s  落点拍 (Landing)     — 留白 / 收束 / 切口
```

Prompt 要**显式**写清这 4 拍各自发生什么，而不是写"一个女生走路"这种持续一致的均匀状态。

### 反例 vs 改写

❌ **反例**（描述持续状态，没有时间结构）：
```
一个穿白裙的女生在落叶中行走，温暖黄昏，温馨感
```

✅ **改写**（4 拍各自交代）：
```
开场：女生背对镜头独自站在梧桐道上，落叶静止悬在半空（0-1s 静帧悬停）。
镜头缓慢推近，落叶开始飘落，她转头侧脸入镜（1-4s 推镜 + 转头）。
风骤起，发丝和裙摆向后扬起，她抬手挡光（4-7s 风扬 + 动作升级）。
她笑了，画面定格在嘴角（7-8s 微笑 + 定帧）。
温暖逆光从树叶间隙透下，丁达尔效应。
```

`★ Insight ─────────────────────────────────────`
**为什么"4 拍"比"一段连续描述"更好用**：Seedance 2.0 在 prompt 解析上对**时间序列描述**比对**状态描述**更敏感。说"开场是 X，1 秒后变 Y，3 秒后变 Z"模型会**真的去构造一个时间过渡**；说"一个 X 在 Y 中"模型只会渲染一帧静态，然后用最小动作填满剩余时间——这就是为什么 AI 默认产出感觉"匀速无情绪"。
`─────────────────────────────────────────────────`

## 运动-情感映射表

每种镜头运动都**默认携带一种情感色彩**——选错了，运动会跟意图打架。下面是对照表，配合 `taste/02-visual-grammar.md` 阅读：

| 情感意图 | 推荐运动 | 中文 prompt | 反向警告 |
|---|---|---|---|
| **好奇 / 揭示** | 缓推 + 定帧 | "镜头从全景缓慢推进到特写，最后定格" | 切勿用"环绕"——会变猎奇感 |
| **紧迫 / 危险** | 快推 + 手持 | "手持镜头快速推进，伴随轻微晃动" | 切勿用"丝滑跟随"——会泄气 |
| **敬畏 / 宏大** | 升镜 + 拉远 | "镜头从地面向上仰拉，主体在画面下方变小" | 切勿用"近景特写"——破坏比例 |
| **温暖 / 平静** | 横移 + 静止 | "镜头从左向右缓慢横移，最后静止在主体上" | 切勿用"剧烈推拉"——破坏 anchor |
| **心痛 / 失去** | 缓拉 + 定帧 | "镜头从特写缓慢后退，主体在画面中越来越小" | 切勿用"环绕"——会变讽刺 |
| **荒诞 / 反差** | 突变切 | "镜头从一个角度突然切到 90 度反向" | 切勿用"丝滑过渡"——平掉笑点 |
| **沉浸 / 主观** | 第一视角手持 | "第一视角手持镜头，前景部分肢体偶尔入画" | 切勿用"第三人称跟随"——失去带入 |
| **悬念 / 停滞** | 静止机位 + 局部动作 | "固定镜头，画面 80% 静止，仅主体某一处缓慢移动" | 切勿加任何 camera 关键词——一加就泄 |

> **铁律**：一段 prompt 里**只许有一个主导运动**。两个运动叠加（"先环绕再推进"）= 模型在两者间漂移，输出糊。要双段运动，拆成两个 clip 再剪辑。

## 首尾帧 vs image2video vs multimodal 的真正选型

`dreamina-mastery.md` 的决策树是按"输入像什么"分类。本节按**"你想钉死什么"**分类——这才是创作维度：

| 你想钉死什么 | 选什么 | 为什么 |
|---|---|---|
| **首帧的精确视觉** | `image2video` | 用 OpenRouter 先生精首帧 → 驱动视频。控制力最强。 |
| **首帧 + 尾帧的精确状态**（变化已知） | `frames2video` | 模型自动插值。适合"从 A 状态过渡到 B 状态"的明确变化。 |
| **特定角色的身份**（不变） | `multimodal2video` + `@image1` 角色 role | 单纯 image2video 容易因运动丢脸。 |
| **特定环境的目的地** | `multimodal2video` + `@image2` 目的地 role | 文字描述"深蓝色宇宙"远不如一张图。 |
| **运镜节奏跟某个参考视频** | `multimodal2video` + `@video1` 运镜 role | 5 秒运镜参考 > 三段 camera prose。 |
| **完全无视觉参考，从零生成** | `text2video` | 但要接受输出会随机；建议**永远先生首帧再 image2video**。 |

### 决策反例

❌ **反例**：用户说"做一个角色穿过门进入另一个空间"。Agent 选 `text2video` 写 200 字 prompt。

**问题**：穿过门 = **首帧（门外）** + **尾帧（门内）** 两个状态，文字描述两次环境是噪声放大；模型很可能给你一个停在门口的画面。

✅ **改写**：
1. OpenRouter 生**门外画面**作为 first frame
2. OpenRouter 生**门内画面**作为 last frame
3. `frames2video` 插值

——精确、可控、不糊。

## Prompt 增强公式（视频版）

借用 `prompt-mastery.md` 的公式，但加上**时间维度**：

```
[质量] + [主体（首帧状态）] + [时间序列动作（4 拍）] + [环境] + [光影] + [运镜] + [节奏]
```

具体填法：

| 维度 | 必填值 | 反例 |
|---|---|---|
| **质量** | `cinematic still`, `professional photography`, `8K detail`, `shot on Arri Alexa` | "高清"（模糊） |
| **主体（首帧）** | 族裔 / 年龄 / 发型 / 服装 / **首帧具体姿态** | "一个女生"（无信息） |
| **时间序列动作** | 0-1s X / 1-4s Y / 4-7s Z / 7-8s 落点 | "走路"（无时间结构） |
| **环境** | 具体地点 + 季节 + 时间 + 关键道具 | "户外"（无锚点） |
| **光影** | 方向 + 色温 + 强度 + 修饰（"黄昏逆光从右侧 45 度，丁达尔效应"）| "温暖光线" |
| **运镜** | **唯一**主导运动（参运动-情感映射表）| "镜头跟随"（默认值） |
| **节奏** | "节奏舒缓 / 张力渐强 / 突变收尾" | 不写 |

## 失败案例对比库

### Case 1: 平庸"咖啡馆女生"

❌ **原 prompt**：
```
镜头跟随，一个穿米色衣服的女生在咖啡馆看书，温暖光线
```

**实际产出**：8 秒匀速跟拍，女生静态翻书，光线均匀。**完全没情感**。

✅ **改写**（情感意图：温暖怀旧，落点是"她合上书的瞬间"）：
```
固定机位中景，一位 25 岁亚洲女性独自坐在咖啡馆窗边，
首帧：她低头阅读一本牛皮笔记本，左手撑着脸颊。
0-2s 静止，只有窗外行人模糊的剪影掠过；
2-5s 她抬头望向窗外，眼神变远，唇角微微上扬；
5-7s 她合上笔记本贴在胸口，靠回椅背；
7-8s 镜头微微拉远定帧，她在画面中略偏右下黄金分割位。
窗外是雨后湿润街道的反光，室内暖钨丝灯，色温 3200K，画面整体偏 Morandi 暖灰。
shot on medium format, shallow depth of field f/2.0, cinematic.
```

**差别**：
- 时间结构（0-2s/2-5s/5-7s/7-8s 各自交代）
- 主体动作有**情绪弧**（专注 → 走神 → 收束 → 安静）
- 光影具体到色温
- 构图位置具体到黄金分割

### Case 2: 反差喜感"猫"

❌ **原 prompt**：
```
一只猫在跳舞，搞笑
```

**问题**：模型不知道"搞笑"是什么；"跳舞"在猫的语境里没有 grounding。

✅ **改写**（情感意图：荒诞反差，落点是"突然停下盯镜头"）：
```
固定低机位特写，一只英短银渐层猫站在木地板上面对镜头。
0-1s 它直立不动；
1-3s 突然开始用后腿小幅交替踏步，节奏类似踢踏舞；
3-5s 节奏加快，前爪同步左右摆动；
5-7s 它做出一个转身动作；
7-8s 突然停下、扭头直视镜头，瞳孔放大，画面定格。
环境为简洁纯白背景，柔和顶光，画面整体偏冷调。
拍摄风格类似宠物纪录片高清特写。
```

**关键技巧**：荒诞情感落在**反差**——前 7 秒是动态行为，最后 1 秒静止凝视。这种"动 → 静"反转必须在 prompt 时间结构里写明，模型才会给。

### Case 3: 紧迫"奔跑"

❌ **原 prompt**：
```
一个人在跑步，紧张感
```

✅ **改写**：
```
手持镜头跟随，第三人称视角。
首帧：一名穿黑色风衣的男性从画面左下角冲入，逆光剪影。
0-2s 他向画面深处奔跑，镜头跟拍但保留一定距离感；
2-5s 镜头加速追上，从背影切换到 3/4 侧脸，能看到他急促的呼吸；
5-7s 他突然侧头回望（没有放慢脚步），表情警觉；
7-8s 镜头脱手般急停，他继续向画面外冲出。
环境：午夜空旷地下停车场，金属冷光从顶部灯管漏下，画面颗粒感明显，
shot on 35mm with handheld shake, color graded cyan-amber teal-orange.
```

**关键技巧**：紧迫感来自三处——**手持晃动**（不是平移）+ **回望**（情绪点）+ **画面外冲出**（未完成感）。三者都要 prompt 显式写。

## 验证 prompt 的最后一道关卡

写完 prompt 跑生成前，用这 5 问拷打自己：

1. ☐ 时间结构有 4 拍吗？还是模糊的"一段时间"？
2. ☐ 第 1 帧和第 8 帧能各自被 freeze 当作独立画面欣赏吗？
3. ☐ 运动是**承载情感**还是**填满时长**？删掉运动看画面还能传达情感吗？删掉了反而更好 → 不要这个运动。
4. ☐ 主体的具体动作有**情绪弧**（变化）还是**单一姿态**（静止状态）？
5. ☐ 光影具体到色温和方向了吗？还是"温暖光线"这种空话？

任何一问答不上 → **回去改 prompt**，不要先跑生成赌运气。生成一次 1-3 分钟，prompt 改一次几秒。
