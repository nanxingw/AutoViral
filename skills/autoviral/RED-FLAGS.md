# Red Flags — 你不能继续的念头

这不是创作指南，是**自检清单**。任何一行只要跟"我刚才正在想…"对上号，**立刻停手**——不是"完成当前一步再回来"，是 **stop mid-token**。每一次让步都在稀释作品。

## 为什么这份单独成文？

`taste/00-prime-directive.md` 讲"应该怎么做"。这份讲"哪些念头是你在偷懒"。两份互补：

- prime-directive = 北极星（定方向）
- red-flags = 烟雾报警器（定阻断）

模式：**"It's fine I'll just X" 是 99% 让步的语言外壳**。

## 创作判断红旗

| 你冒出的念头 | 真正含义 | 立刻做什么 |
|---|---|---|
| "这个调性挺像 X 平台用户喜欢的" | 你在用**集体人格**替代具体情感判断 | 回 `taste/00`，问"我想让一个具体观众感到什么单一情感" |
| "前 3 秒先拍点能抓眼球的就行" | 你在执行平台 SOP | 回 `taste/00` 第 18 行，"3 秒法则是广告投放指标，不是创作法则" |
| "AI 默认就是这样，先这样吧" | 你接受了**平庸合格品** | 回 `taste/06-rubric.md`，预评，不达 28 重做 |
| "这一刀剪在哪都行，反正过得去" | 你在**匀速剪辑**，没有情感节奏 | 回 `taste/03`，问"这一刀情感上为什么是这一刀" |
| "用户喜欢就行" | 模糊判断 | 具体回答"用户的什么动机被这个作品满足了" |
| "数据涨了就是好作品" | 用平台指标定义品味 | 数据 ≠ 品味；3 个月后还会被想起的才是 |
| "之前这么剪都过了" | 用习惯替代独立判断 | 不要复用旧模板，对当前作品独立想 |
| "差不多了，就这样吧" | 偷懒 | 想象作品发出去后是否会被分享，no → 重做 |

## 信息不足时的红旗

| 你冒出的念头 | 真正含义 | 立刻做什么 |
|---|---|---|
| "用户没说那么细，先这样" | 用沟通缺口替代品味缺口 | **主动反问**情感意图（"想让观众前 3 秒感到什么？"），不要默默猜 |
| "用户说看着办，那我猜吧" | 默默决定 = 黑箱 | 做有根据的猜测，**明确告诉用户你猜了什么**，等纠正 |
| "感觉用户想要 X" | 投射 | 回到对话原文，引用用户实际说的话 |

## Prompt 工程红旗

| 你冒出的念头 | 真正含义 | 立刻做什么 |
|---|---|---|
| "这个 prompt 看起来差不多了" | 在希望 AI 帮你想 | 用 `assets/capabilities/prompt-mastery` 的增强公式，**每一个**维度（quality/subject/lighting/camera/style/color）都填具体值 |
| "这个不需要 reference，AI 会画" | 赌默认值，结果会平庸 | `frame-gacha` 多生候选 + `reference-directives` 锁定 |
| "中景居中应该够了" | 接受 AI 默认构图 | AI 默认就是中景居中；要打破必须**显式覆盖**（"low angle / negative space on right / 黄金分割构图"）|
| "一个 prompt 多塞几个意图，模型会自己选" | 寄希望于模型解歧义 | 不会。多意图 = 输出糊。一帧一构图，多意图拆 `multimodal2video` + role 分配 |
| "首帧用 text2video 应该差不多" | 默认值赌博 | 首帧**永远**先用图片生成精修，再用 `image2video` 驱动 |

## 工具决策红旗

| 你冒出的念头 | 真正含义 | 立刻做什么 |
|---|---|---|
| "这个工具最近没成功过，但试一次也没事" | 不做状态检查 | 批量前**必须** `check_providers.py` |
| "降级一下就好" | 静默降质 | 任何降级**必须告知用户并获得确认**——参 `assets/capabilities/fallback-strategy.md` |
| "这个工具失败了，让我试别的" | 不诊断根因 | 看 error log；签名拒绝走 `filter-retries`，配额走 `fallback-strategy`，工具崩走 `check_providers` |
| "字幕用 ffmpeg drawtext 拼一下吧" | 跳过封装 | **必须用** `subtitle_burn.py`——封装了字体管理、安全区、中英混排，手写极易出低级事故 |

## 节奏与情感红旗

| 你冒出的念头 | 真正含义 | 立刻做什么 |
|---|---|---|
| "音乐有节拍我就 beat-sync 一下" | 让工具替代思考 | beat-sync 是工具，**哪一刀切在 beat 上、哪一刀切在反拍**由 `taste/03` 决定 |
| "这段加点音乐就完整了" | 用音乐遮盖叙事缺口 | 回 `taste/01`，问"叙事弧的哪一环缺了"——补叙事，不是补音乐 |
| "字幕加上观众就懂了" | 用字幕替代视觉表达 | 字幕是辅助，主表达走画面；如果画面表达不出，重生画面 |
| "payoff 没那么强，但整体节奏顺" | 缺高潮的"顺"是平的 | 回 `taste/01`，明确 payoff 瞬间，配 `taste/03` 的停顿/drop/特写**显式标记** |

## 交付前最后一道红旗

| 你冒出的念头 | 真正含义 | 立刻做什么 |
|---|---|---|
| "评分 3 分应该能交付吧" | 想偷过 rubric | rubric < 3.5 **不交付**——参 `taste/06`。发出去就是稀释创作者账号 |
| "用户在等了，先发吧" | 用 deadline 替代品味 | 告知用户"目前评分 3 分，需要 X 时间到 3.5"。让用户决定是否要差一档版本 |
| "这个 work 不重要，质量低点没事" | 没有"不重要的 work" | 每个 work 都进同一个账号——平均拉低就是拉低 |

## 元规则

> **Rule of Thumb**：当你读到这份清单的某一行，发现"啊我刚想这么做"——**立刻**停下来，回 `taste/` 重思。
>
> 不要"也算合理化继续"。每一次让步都在稀释作品。

---

## 这份清单怎么用

1. **每次开工前** — 心里默扫一遍清单。已经内化的话不用再读。
2. **遇到 friction 时** — 当你犹豫"要不要这样办"，搜清单看是不是命中红旗。
3. **交付前** — rubric 自检之前，红旗扫一遍。
4. **被纠正后** — 用户纠正后，回看清单，识别是哪个红旗没拦住，**写下来**（增加新条目）。

清单本身**会进化**——发现新的 rationalization 模式就加进来。文件轻量，rigid。
