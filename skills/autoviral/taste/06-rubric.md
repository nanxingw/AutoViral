# The Rubric

## 这份文件的使命

所有的 taste、所有的镜头语法、所有的节奏知识，最后都落到**一件事**：

> 这件作品**为什么**值得发出去？

答不上来 = 不发。发了 = 稀释创作者的账号品味。

这份 rubric 不是事后打分，是**产出前的最后一道门**。每次交付前（给用户看之前），你自己先对着过一遍。

## 评分维度（8 项，每项 1-5 分）

### 1. 钩子强度（Hook Strength）
**问**：前 3 秒是否建立了一个让观众无法划走的承诺？

| 分 | 表现 |
|---|---|
| 1 | 慢热开场、自我介绍、"今天给大家分享" |
| 2 | 有钩但平庸（标题式陈述、常见话术） |
| 3 | 合格的钩子，但观众看过类似的 |
| 4 | 强钩——反常识 / 视觉冲击 / 具体情境 |
| 5 | 前 2 秒产生强烈"我必须看完" |

### 2. 情感清晰度（Emotional Clarity）
**问**：这个作品的**主导情感**是一个词能说清的吗？观众的身体反应会是这个情感吗？

| 分 | 表现 |
|---|---|
| 1 | 说不清想让观众感到什么 |
| 2 | 情感模糊或多个混杂 |
| 3 | 主导情感清晰，但表现一般 |
| 4 | 情感集中、画面/剪辑/声音都在为它服务 |
| 5 | 情感足够强，看完 5 分钟后观众还记得 |

### 3. 叙事完整度（Narrative Completeness）
**问**：有没有完整的情感弧？有没有可辨识的 payoff？

| 分 | 表现 |
|---|---|
| 1 | 没有结构，镜头堆砌 |
| 2 | 有开头无结尾 / 有回报无铺垫 |
| 3 | 结构完整但节奏平 |
| 4 | 清晰的弧，payoff 被标记出来 |
| 5 | 结构巧妙，payoff 有记忆点 |

### 4. 视觉语法质量（Visual Grammar）
**问**：每个镜头的大小、机位、运动、构图是为情感服务的吗？有没有废镜头？

| 分 | 表现 |
|---|---|
| 1 | 大量"填充镜头"、匀速中景、居中证件照 |
| 2 | 画面漂亮但没语言 |
| 3 | 基础正确，偶有亮点 |
| 4 | 多数镜头在说话，有明确视觉节奏 |
| 5 | 每个镜头都有必要性，视觉语法精准 |

### 5. 节奏与剪辑（Rhythm & Editing）
**问**：节奏有起伏吗？切点服务于情感吗？有匀速/废拍吗？

| 分 | 表现 |
|---|---|
| 1 | 匀速、过渡滥用、音画脱节 |
| 2 | 节奏单调、切点随意 |
| 3 | 节奏合理，无明显错误 |
| 4 | 节奏有设计，切点有意图 |
| 5 | 节奏本身成为表达手段 |

### 6. 设计与信息（Design & Legibility）
**问**：视觉层级清晰吗？文字清楚可读吗？配色统一吗？

| 分 | 表现 |
|---|---|
| 1 | 层级混乱、文字看不清、配色打架 |
| 2 | 能看懂但粗糙 |
| 3 | 清晰合格 |
| 4 | 设计讲究，视觉世界统一 |
| 5 | 设计本身是一个可被记住的标志 |

### 7. 声音设计（Audio Craft）
**问**：音乐是否匹配情感？关键瞬间有没有被声音标记？

| 分 | 表现 |
|---|---|
| 1 | 随便配的 BGM 或无声音设计 |
| 2 | 音乐合适但无设计 |
| 3 | 有基础的音画同步 |
| 4 | 音乐驱动情感，关键瞬间有声音强调 |
| 5 | 声音成为独立的叙事层 |

### 8. 可分享性（Shareability）
**问**：这个东西如果出现在我自己的关注流里，**我会分享给朋友吗**？

| 分 | 表现 |
|---|---|
| 1 | 不会，看过就忘 |
| 2 | 可能点赞，不会分享 |
| 3 | 会收藏，不会主动发 |
| 4 | 会发给一两个特定的人 |
| 5 | 会发到群里 / 发朋友圈 |

## 使用规则

### 交付门槛

**总分低于 28**（平均 3.5）的产出**不要交付给用户**。

回到最薄弱的两项（1-2 分的项），想清楚怎么改，再重做那一环。

### 诚实打分

给自己打分最大的陷阱是**自我安慰**。有两个测试帮助诚实：

1. **盲评测试**：把分数的依据逐条写出来，每条要有具体证据（"第 3 秒镜头大小是中景，情感意图是敬畏，不匹配——视觉语法 2 分"）。说不出证据的 = 虚高。
2. **反面测试**：这件作品如果是别人做的，我会打多少分？如果答案比自评低 1 分以上，说明我在护自己的作品。

### 对用户展示

给用户反馈时，**不要只给总分**。给具体的：

- 哪两项最薄弱
- 薄弱的具体表现是什么（指出具体时间点或画面）
- 怎么改可以把它从 X 提到 X+1

**诚实 > 礼貌**。用户需要的是能提高的反馈，不是"不错哦"。

## 与 `taste/05-creative-schema` 的关系

Schema 告诉你**做什么**，Rubric 告诉你**做得好不好**。二者配合：

1. 开工前填 schema → 知道目标
2. 产出后跑 rubric → 检查是否达成目标
3. 不达成 → 回到 schema 的哪一项没被执行

## 禁止项

- ❌ **不要**把"观众数据好"作为评分证据。这是追随者逻辑，不是创作者逻辑。有品味的作品可能冷门，烂作品可能爆红——rubric 评的是作品本身，不是流量。
- ❌ **不要**把"符合平台调性"作为加分项。平台调性是创作的限制条件，不是质量标准。
- ❌ **不要**为节省时间跳过 rubric。rubric 如果感觉烦，就是它最发挥作用的时候。
