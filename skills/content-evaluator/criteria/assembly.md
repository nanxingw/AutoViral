# 合成阶段评审标准

## 评审维度

### 1. 技术参数 (technical_params)
检查方法：使用 ffprobe 检查最终输出

```bash
# 必须运行的检查命令
ffprobe -v error -show_entries format=duration,size -show_entries stream=width,height,r_frame_rate,codec_name,codec_type,sample_rate,channels -of json output/final.mp4
```

**检查项：**
- 分辨率：竖屏1080x1920，横屏1920x1080
- 编码：H.264 (libx264)
- 帧率：≥24fps（推荐30fps）
- 音频：AAC，44100Hz
- 时长：符合方案规划（±20%误差范围内）
- 文件大小：合理范围内（不超过100MB/分钟）
- 是否有音频轨（必须有！）

**评分标准：**
- 9-10: 所有技术参数完美达标
- 7-8: 基本达标，个别参数可优化
- 5-6: 有技术问题但不影响播放
- 1-4: 严重技术问题（无法播放、无音频、分辨率错误）

### 2. 剪辑节奏 (editing_rhythm)
- 镜头切换频率是否与内容类型匹配？
- 是否有不合理的长静止镜头？
- Hook部分（前3秒）的节奏是否紧凑？
- 结尾是否有适当的收束？

**评分标准：**
- 9-10: 节奏感极佳，完播率预期高
- 7-8: 节奏合理，没有明显的拖沓
- 5-6: 有部分段落节奏失衡
- 1-4: 整体节奏混乱或极度拖沓

### 3. 音画协调 (audio_visual_sync)
- BGM风格是否与内容氛围匹配？
- BGM音量是否合适（不压过人声）？
- 音乐是否有淡入/淡出处理？
- 如果是卡点视频，画面切换是否与节拍对齐？
- 是否有突兀的音频断裂或杂音？

**评分标准：**
- 9-10: 音画完美配合，音乐增强了视觉体验
- 7-8: 音画协调，BGM选择得当
- 5-6: 有音乐但匹配度一般
- 1-4: 音画严重不协调或无BGM

### 4. 字幕质量 (subtitle_quality)
- 字幕是否与画面同步？
- 字幕位置是否合理（不遮挡主体）？
- 字幕是否清晰可读（大小、颜色、描边）？
- 内容是否准确、无错别字？
- 每行字数是否合理（15-20字以内）？

**评分标准：**
- 9-10: 字幕专业精美，增强了内容表达
- 7-8: 字幕清晰可读，时间同步
- 5-6: 字幕存在小问题（位置不佳、偶有不同步）
- 1-4: 字幕严重影响观看体验

### 5. 调色一致性 (color_grading)
- 全片色调是否统一？
- 是否有应用调色（不是原始色调）？
- 调色是否适合内容类型和平台？
- 不同片段之间的色彩过渡是否平滑？

**评分标准：**
- 9-10: 调色专业，统一的视觉风格
- 7-8: 有基础调色，色调基本统一
- 5-6: 调色不一致或无调色
- 1-4: 色调混乱，各片段色彩差异巨大

### 6. 发布文案 (publish_text)
- 标题是否吸引人（含关键词、有吸引力）？
- 正文是否完整（含平台特定格式）？
- 标签/话题是否合理（数量、相关性）？
- 发布时间建议是否合理？

**评分标准：**
- 9-10: 文案专业，SEO优化到位，话题精准
- 7-8: 文案完整，基本优化
- 5-6: 文案存在明显缺项
- 1-4: 缺少发布文案或质量极差

## Critical 问题示例
- 最终视频无音频轨
- 最终视频无法播放（编码错误）
- 画面出现黑边或黑屏段落
- 时长严重偏离方案（误差>50%）
- 缺少发布文案（publish-text.md不存在）
- 视频中出现了不属于此内容的画面（素材错误）
